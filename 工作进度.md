## 各阶段工作详情

### 阶段一：阅读设计文档

- 阅读 `docs/MVP服务端开发与测试设计.md` 与 `docs/服务端架构与功能设计.md`。
- 理清核心约束：服务端权威模拟、Tick 边界结算、客户端不可直接改状态。
- 确认 MVP 范围：单战场、单星球平面地图、固定玩家 key 鉴权。

### 阶段二：项目骨架搭建（M0）

**完成内容：**

1. **创建 Go 模块**：`module siliconworld`，依赖 `gopkg.in/yaml.v3`。
2. **配置层** (`internal/config/config.go`)：
   - YAML 配置解析，支持玩家列表、地图参数、Tick Rate、端口、限流配置。
   - 提供 `KeyToPlayer()` 方法将 Bearer Key 映射为 PlayerID。
3. **数据模型**：
   - `model/entity.go`：Building（base/mine/solar_plant/factory/turret）和 Unit（worker/soldier）实体定义，包含默认属性和建造/生产费用。
   - `model/command.go`：CommandRequest、CommandResponse、QueuedRequest 及所有错误码定义。
   - `model/event.go`：GameEvent、EventType 枚举定义。
   - `model/world.go`：WorldState（含 sync.RWMutex）、MapTile、PlayerState、资源模型，提供坐标工具函数。
4. **HTTP 服务**：配置路由、鉴权中间件、健康检查接口。
5. **Tick Loop 骨架**：GameCore 启动 goroutine，固定频率触发 processTick。

### 阶段三：命令执行闭环（M1）

**完成内容：**

1. **命令队列** (`internal/queue/queue.go`)：
   - 线程安全的 `Enqueue`/`Drain` 操作。
   - 基于 `request_id` 的去重（seen map），重复提交直接拒绝。
2. **六类命令规则** (`internal/gamecore/rules.go`)：
   - `build`：消耗矿物/能量，在指定坐标创建建筑，检查边界/占位。
   - `move`：单位移动，检查所有权、移动范围、目标地块合法性。
   - `attack`：单位攻击单位或建筑，计算伤害，销毁 HP≤0 实体。
   - `produce`：在工厂生产单位，消耗资源，自动寻找相邻空格生成。
   - `upgrade`：升级建筑，按等级倍数收费，刷新建筑属性。
   - `demolish`：拆除建筑，返还 50% 资源，不允许拆自己的基地。
3. **Tick 结算**：每 Tick 依次执行命令 → 资源结算 → 炮塔自动攻击 → 胜负判定。
4. **幂等性**：同一 `request_id` 在 Gateway 层即被拒绝，不重复进入队列。

### 阶段四：最小玩法闭环与迷雾裁剪（M2）

**完成内容：**

1. **迷雾引擎** (`internal/visibility/visibility.go`)：
   - `VisibleTiles`：根据玩家所有建筑和单位的视野范围（圆形范围）计算可见格子集合。
   - `FilterBuildings`/`FilterUnits`：对查询结果做迷雾裁剪，敌方实体仅在可见格显示。
   - `FilterEvent`：SSE 事件按 `visibility_scope` 过滤（`all`/`player_id`）。
   - `FogMap`：返回二维 bool 可见性网格。
2. **查询层** (`internal/query/query.go`)：
   - `GET /state/summary`：玩家资源（仅自己可见），所有玩家存活状态，当前 Tick，地图尺寸。
   - `GET /galaxy`：星系概览（MVP：单系统）。
   - `GET /systems/{system_id}`：系统概览（MVP：单星球）。
   - `GET /planets/{planet_id}`：带迷雾裁剪的星球详情（建筑+单位）。
   - `GET /fogmap`：玩家视野网格。
3. **资源结算**：每 Tick 对所有激活建筑产出矿物/能量，能量不足时建筑停工。
4. **炮塔自动攻击**：每 Tick 内炮塔扫描攻击范围内最近敌方单位。
5. **胜负判定**：玩家失去最后一座基地即视为出局，最后存活玩家获胜。
6. **持久化** (`internal/persistence/persistence.go`)：审计日志追加记录，快照异步落盘 JSON。

### 阶段五：SSE 事件推送

**完成内容：**

1. **EventBus**：订阅者 map（subscriber_id → channel），Publish 向所有订阅者广播。
2. **SSE Handler**：
   - 检测 `http.Flusher` 支持，设置 `text/event-stream` 头。
   - 每次接收事件后经 Visibility Engine 过滤再推送。
   - 连接断开自动 Unsubscribe，channel 关闭。
3. **慢订阅保护**：channel 缓冲 256，满时丢弃，不阻塞 Tick。

## API 契约

### 鉴权

```
Authorization: Bearer <player_key>
```

### 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /health | 健康检查（无需鉴权）|
| GET | /metrics | 运行指标（无需鉴权）|
| GET | /state/summary | 世界摘要（含当前玩家资源）|
| GET | /galaxy | 星系概览 |
| GET | /systems/{system_id} | 恒星系概览 |
| GET | /planets/{planet_id} | 星球详情（含迷雾裁剪）|
| GET | /fogmap | 玩家视野网格 |
| POST | /commands | 提交命令批次 |
| GET | /events/stream | SSE 事件流 |

### 命令类型

| 命令 | 必填参数 |
|------|---------|
| build | target.position, payload.building_type |
| move | target.entity_id, target.position |
| attack | target.entity_id, payload.target_entity_id |
| produce | target.entity_id（工厂ID）, payload.unit_type |
| upgrade | target.entity_id |
| demolish | target.entity_id |

### 错误码

| 错误码 | 含义 |
|--------|------|
| OK | 成功 |
| INVALID_TARGET | 目标不合法 |
| NOT_OWNER | 操作非己方实体 |
| OUT_OF_RANGE | 超出范围 |
| INSUFFICIENT_RESOURCE | 资源不足 |
| DUPLICATE | 重复 request_id |
| VALIDATION_FAILED | 格式或字段校验失败 |
| ENTITY_NOT_FOUND | 实体不存在 |
| POSITION_OCCUPIED | 目标位置已被占用 |

## 已知限制与 Post-MVP 待做项

- 当前 `GET /events/stream` 不支持断线重连（无 Last-Event-ID 恢复机制）。
- 回放工具尚未实现（命令日志已落盘，重放执行器待开发）。
- 持久化快照仅在收到显式触发时落盘，暂无定时快照调度。
- 移动命令为瞬移（单 Tick 到达），尚未实现多 Tick 路径追踪。
- 科技树、跨恒星系功能属于 Out-of-Scope，不在本 MVP 内。
