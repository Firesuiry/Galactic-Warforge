# 命令事件与可见性系统设计

## 1. 设计目标
- 将所有玩家行为统一为命令输入，确保规则可校验、可回放。
- 将系统变化统一为事件输出，支持实时推送与审计分析。
- 按玩家视野与权限裁剪查询和事件内容。

## 2. 命令系统
- 命令字段：
  - `type`
  - `issuer_type/issuer_id`
  - `target`
  - `payload`
  - `at_tick`
- 执行流程：
  - 接收并鉴权
  - 参数校验与权限校验
  - 入队并在 Tick 内执行
  - 返回命令结果码与描述

## 3. 事件系统
- 事件字段：
  - `event_id`
  - `event_type`
  - `tick`
  - `scope`
  - `payload`
- 事件来源：
  - 生产与物流变化
  - 战斗结算
  - 科技完成
  - 戴森球与能源变化
  - 审计与告警

## 4. 可见性系统
- 输入：
  - 玩家阵营关系
  - 探测设施与半径
  - 战争迷雾状态
- 输出：
  - 视野内实体查询结果
  - 视野内事件流
- 原则：不可见信息不下发，不允许通过事件侧信道反推完整敌情。

## 5. 回放与审计
- 回放基于命令日志和关键状态摘要重建过程。
- 审计记录覆盖请求来源、执行结果、关键参数与影响范围。
- 回滚依赖快照点与命令重放，确保一致性与可追责。
