# 戴森球系统设计

## 1. 设计目标
- 支持从太阳帆轨道发射到戴森球结构建造的完整流程。
- 将戴森系统纳入 Tick 结算，稳定产出可用能量与高阶资源链输入。

## 2. 核心对象
- 太阳帆：有寿命、轨道、发射与衰减状态。
- 戴森层：按恒星组织的多层结构容器。
- 节点/框架/壳层：构成戴森球结构单元。
- 应力参数：限制结构扩张速度与稳定性。

## 3. 建造链路
- `em_rail_ejector` 发射太阳帆形成戴森蜂群。
- `vertical_launching_silo` 发射火箭用于节点/框架/壳层建造。
- 建造行为受资源、能源、科技和建造队列约束。

## 4. 结算规则
- 太阳帆每 Tick 产生轨道能量并进行寿命衰减。
- 戴森结构完成度提升后增加稳定产能。
- 产能分配去向：
  - 射线接收站发电
  - 临界光子链路
  - 终局能源与制造链

## 5. 异常与保护
- 应力超限：限制新结构挂载并触发告警。
- 供料中断：建造进度冻结，保留任务状态。
- 能量过载：按策略限流到非关键负载，避免主网崩溃。
