# 战斗与防御系统设计

## 1. 设计目标
- 建立敌我单位、探测、攻击、防御、告警的统一战斗闭环。
- 支持行星防御与轨道层冲突，服务端权威判定伤害与结果。

## 2. 战斗对象
- 我方：
  - 防御建筑（机枪塔、导弹塔、激光塔等）
  - 侦测与指挥设施（信号塔、战场分析基站）
  - 行星护盾
- 敌方：
  - 黑雾相关敌对单位与威胁实体
  - 分层威胁等级与刷新/进攻行为

## 3. 战斗流程
- 侦测阶段：更新目标可见状态与威胁列表。
- 锁定阶段：依据射程、目标类型、优先级选择攻击目标。
- 结算阶段：计算命中、伤害、护甲/护盾抵消与单位状态变化。
- 事件阶段：发布受击、击毁、护盾告警、波次变更等事件。

## 4. 防御建筑职责
- 高斯/导弹/聚爆/激光/电浆炮塔：承担不同目标类型与距离段火力。
- 干扰塔：降低敌方效率或速度，提供控制收益。
- 信号塔：增强战术信息覆盖与协同射击。
- 行星护盾：吸收来自高威胁目标的伤害，保护核心产线。

## 5. 与其他系统联动
- 能源系统：防御设施优先供电策略直接影响生存能力。
- 物流系统：弹药/维护物资供给影响持续作战能力。
- 科技系统：武器伤害、射程、护盾容量、探测半径由科技提升。
