# 物流与生产系统设计

## 1. 设计目标
- 打通“采集 -> 运输 -> 加工 -> 仓储 -> 配送”的完整工业闭环。
- 支持行星内与星际两级物流网络。
- 在 Tick 中保证吞吐、库存与任务状态一致。

## 2. 物流网络分层
- 局部层：传送带、分拣器、分流器、堆叠器构成建筑间微循环。
- 站点层：行星物流站负责本地供需匹配与无人机调度。
- 星际层：星际物流站负责跨星球订单与运输舰调度。

## 3. 物流站模型
- 核心字段：
  - 站点库存、槽位配置、供需模式、功耗状态
  - 调度队列、无人机/运输舰数量与状态
  - 目标站点、在途任务、异常状态
- 供需模式：
  - 供应（Supply）
  - 需求（Demand）
  - 存储（Storage）

## 4. 生产模型
- 配方定义：
  - 输入物品集合
  - 输出物品集合
  - 周期 Tick 与能耗
  - 科技解锁要求
- 生产状态：
  - 当前配方、进度、堵塞原因、空转标记
  - 输入缓存/输出缓存、喷涂增益状态

## 5. Tick 结算顺序建议
- 读取电力可用预算，判定停机与降载。
- 处理采集产出，写入局部缓存。
- 处理传送带与分拣搬运，推进站点收发。
- 执行站点调度与跨站分配任务。
- 推进生产进度并结算产物。
- 发布物流与生产事件，更新统计与告警。

## 6. 异常处理
- 断电：建筑停机，进度冻结或回退策略按类型执行。
- 堵塞：输出满导致上游停滞，触发拥塞告警事件。
- 断链：站点不可达时保留任务并按策略重试。
