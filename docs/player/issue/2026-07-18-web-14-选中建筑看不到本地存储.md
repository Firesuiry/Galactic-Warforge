## 修复内容（2026-07-18）

1. 迷你浮层（`PlanetSelectionBar.tsx`）：选中建筑时信息区新增本地存储摘要行（"库存 硅矿 12 · 容量 12/20"，物品名走 catalog/i18n 翻译，容量取 `building.storage.capacity` 回退 `runtime.functions.storage.capacity`）；己方建筑操作区新增"详情"入口（`onShowDetail`）。
2. `PlanetPage.tsx`：点选建筑/单位时除自动滑出抽屉外，同步切到"选中对象"页签（与深链 `?select=` 行为一致）；"详情"按钮同样跳转。
3. 侧栏"库存与任务"区（`PlanetPanels.tsx` 新增 `BuildingStorageSection`）：原始 JSON dump 改为结构化 kv 列表——本地存储（物品/数量/总量/容量）、输入/输出缓存、产出速率（采集建筑 yield_per_tick）、生产配方（翻译名 + 剩余 tick）、当前任务；库存总量 ≥ 容量 90% 时显示"存储将满"警示。
4. `shared-client/types.ts`：`Building.storage` 类型补齐 capacity/slots/buffer_capacity 字段（与服务端 StorageState 一致）。
5. 测试：`PlanetSelectionBar.test.tsx` 新增库存摘要与详情回调用例；`PlanetPanels.test.tsx` 新增结构化存储区/将满警示/空存储用例；Playwright `tests/planet-hud-resources.spec.ts` 在真实服务器上验证选中 b-1 后迷你条容量摘要与侧栏本地存储展示。

## 问题描述

在地图上选中 `mining_machine` 后，底部只弹出"采矿机 (5,1) 运行中 + 升级/拆除"的迷你浮层，看不到建筑本地存储内容。当前经济模型下矿机产出全部进建筑本地存储（见 gameplay-02），玩家无法知道矿机里已经积了多少硅矿、是不是满了、为什么 minerals 没涨。

## 实际现象

- 选中矿机：仅 名称/坐标/运行状态 + 升级/拆除 按钮
- `inspect` API 里建筑 runtime 有 storage 定义，但 GUI 选中态不展示存储数量、产出速率、积压情况
- 物流站有"详情"页签（文档提到），普通生产/采集建筑没有等价视图

## 推荐改进

1. 选中任意建筑时在侧栏"详情"页签展示：本地存储（物品/数量/容量）、运行状态与原因、能耗、产出速率
2. 采集建筑明确显示"已采硅矿 ×N / 容量 M"，积压将满时给提示
3. 迷你浮层保留快捷操作（升级/拆除），但加一个"详情"入口
