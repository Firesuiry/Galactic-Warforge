# 行星页初始视角全图漆黑找不到基地，"回家"按钮不定位基地

- 状态：已修复
- 首次记录：2026-07-18
- 来源：2026-07-18 Web 试玩
- 类别：Web / 行星地图 / 新玩家引导

## 问题描述

新玩家第一次打开行星页（`/planet/planet-1-1`）时，画布以 8px/tile 的缩放展示 2000x2000 地图，画面几乎全黑（迷雾+超小图块），看不到自己的基地在哪。左侧工具栏的 ⌂（回家）按钮也不定位到基地，只是把相机重置到地图原点（offset 32,32 @ 8px），基地被挤在视口左上角边缘，依然不可辨认。

## 实际现象

- 初始视角截图：整屏黑点，玩家无任何"我在这里"的锚点
- ⌂ 点击后 `data-camera-offset-x/y = 32,32`，`tileSize = 8`：基地 (3,3) 距视口边缘仅 ~50px，8px 图块看不清
- 玩家必须自己滚轮放大 + 拖拽才能找到基地，而这正是最需要引导的开局 5 分钟

## 推荐改进

1. 首次进入行星页时相机直接以玩家基地（或执行体）为中心，默认缩放到 24~32px/tile
2. ⌂ 按钮语义改为"回到基地"：居中 + 合理缩放，而不是回地图原点
3. 小地图（右下角）上高亮基地位置和当前视口范围，点击可跳转

## 证据

- 截图：`docs/player/assets/2026-07-18-web-playtest/10-planet-fresh.png`（初始视角）、`12-zoomed.png`（手动放大后）
- 实测相机参数：home 后 `offset 32,32 @ 8px`

## 修复记录（2026-07-18，提交：f54bd9e）

- `model.ts` 新增 `resolveHomeTile(planet, playerId)`：第一个自有建筑（按 id 排序）为"家"，无建筑退回第一个自有单位。
- `store.ts`：`FocusRequest` 支持可选 `zoomIndex`；新增 `PLANET_HOME_ZOOM_INDEX = 8`（32px/tile）。
- `PlanetPage.tsx`：首次进入行星页（URL 无 x/y 深链）自动 `requestFocus(home, PLANET_HOME_ZOOM_INDEX)`，相机直接居中基地、32px/tile。
- `PlanetMapPixi.tsx`：聚焦 effect 应用 focus 请求携带的缩放档。
- `PlanetMapToolbar.tsx` / `PlanetPanels.tsx`：⌂ 与"重置视角"按钮语义改为"回到基地"（居中基地 + 32px/tile），找不到基地时退回原重置行为。
- e2e `planet-build-workflow.spec.ts` 新增"首次进入行星页视角聚焦基地"用例（断言 32px/tile、基地在视口内、可点选）。
