# /war 蓝图工作台只有"从底盘新建"，缺少"基于公开预置蓝图创建变体"入口

- 状态：已修复（2026-07-18，提交 hash：b2742f3）
- 首次记录：2026-07-18
- 来源：2026-07-18 Web 试玩（config-war 官方战争场景）
- 类别：Web / 战争 / 蓝图

## 修复内容

- `client-web/src/features/war/components/forms/BlueprintVariantForm.tsx`：父蓝图下拉合并 `catalog.warfare.public_blueprints`（映射为预置父本）与玩家自有蓝图，按「公开预置蓝图 / 我的蓝图」optgroup 分组；删除"玩家 0 蓝图时整表单隐藏"的空态，开局即可选预置父本派生变体（server 端 `blueprint_variant` 本已接受预置父本 ID，无需 server 改动）。
- `client-web/src/pages/WarPage.tsx`：「蓝图选择」为空时显示引导文案（去蓝图改型派生变体）；「创建蓝图」卡片补充变体入口引导。
- `client-web/src/pages/WarPage.test.tsx`：新增回归——blueprints 为空 + catalog 带 public_blueprints 时，改型表单可见、默认选中预置父本、提交 `blueprint_variant` 且 `parent_blueprint_id` 为预置 ID。
- 新增 Playwright 回归 `client-web/tests/war-blueprint-variant-preset.spec.ts`（真实浏览器 + 战争服），已通过。

## 问题描述

`docs/player/玩法指南.md` 推荐的战争验证路径第一步是 `blueprint_variant corvette corvette_verify utility --name ...`，即以公开预置蓝图为父本创建变体。但 `/war` 页的蓝图工作台"创建蓝图"表单只有：蓝图 ID / 蓝图名称 / 作战域（ground/air/orbital/space）/ 底盘（corvette_hull 等 5 种 hull），没有"选择公开预置蓝图作为父本"的入口。"蓝图选择"下拉在开局时也是空的。

## 影响

- 玩家按文档在 GUI 里走战争闭环时第一步就对应不上：只能创建空壳蓝图，之后还要逐个 `blueprint_set_component`，GUI 里是否有完整的组件配置/校验/定稿链路尚未验证
- 战争主线的 GUI 可达性明显落后于 CLI

## 推荐改进

1. 蓝图创建表单增加"从预置蓝图创建变体"模式：列出 `catalog.warfare.public_blueprints`，选父本 + 填变体 ID/名称即可
2. 创建后引导下一步（配置组件 → 校验 → 定稿），各步在当前蓝图面板里给出状态标记
3. 把"蓝图变体 → 校验 → 定稿 → 排产 → 编成舰队"做成一条 GUI e2e 回归

## 证据

- 试玩实测表单字段：`input ×2（ID/名称）+ select ×2（作战域/底盘）+ 创建蓝图按钮`，公开预置蓝图未出现在任何下拉里
