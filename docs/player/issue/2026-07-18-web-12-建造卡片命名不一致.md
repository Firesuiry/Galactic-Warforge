# 建造栏卡片命名不一致："采矿机"用中文名，其余用英文 ID

- 状态：已修复
- 首次记录：2026-07-18
- 来源：2026-07-18 Web 试玩
- 类别：Web / 建造栏 / 文案一致性
- 修复提交：b2742f3

## 修复内容（2026-07-18）

根因与 issue 最初判断不同：服务端 catalog 的 `name` 已全部统一为英文（mining_machine 也是 "Mining Machine"），混用来自前端字典缺口——`translation-config.ts` 的 buildingType 字典只有 7 个键（且 assembler/smelter/storage/power_generator 四个是与服务端 ID 不匹配的陈旧死键），未命中的建筑原样显示下划线裸 ID。

1. `translation-config.ts`：buildingType 字典补齐服务端 `building_defs.go` 全部 62 个建筑 ID 的中文名，删除 4 个陈旧死键。
2. `translate.ts`：`translateCatalogBackedValue` 在字典未命中且 catalog 提供英文名时回退到英文名（如 "Wind Turbine"），不再暴露 snake_case 裸 ID；未来新增建筑不会再出现裸 ID。
3. `fixtures/scenarios/baseline.ts`：fixture catalog 建筑名改为与服务端一致的英文名（Mining Machine 等），让测试真实覆盖字典翻译路径。
4. 测试：新增 `i18n/translate.test.ts`（回退链与字典完整性）；`PlanetBuildBar.test.tsx` 新增"卡片名不得为 snake_case 裸 ID"断言；e2e `planet-build-workflow.spec.ts` 建造提示语断言同步更新为中文名。

## 问题描述

底部建造栏卡片的 title 大部分是英文建筑 ID（`wind_turbine` / `matrix_lab` / `tesla_tower` / `conveyor_belt_mk1`），唯独 mining_machine 显示中文名 `采矿机`。玩家看到的列表是：`wind_turbine · matrix_lab · tesla_tower · 采矿机 · conveyor_belt_mk1 ...`。

## 影响

- 视觉上不统一；对依赖英文 ID 对照文档/CLI 的玩家造成困惑（文档里写 `build 5 1 mining_machine`，界面上找不到 mining_machine）
- 自动化/e2e 也容易踩坑（本次试玩按 `title^="mining_machine"` 选卡片就找不到）

## 推荐改进

1. 卡片统一显示"中文名 + 英文 ID"两行（或中文名为主、title 提示 ID），从 catalog 的 `name` 字段取，不要中英文混用
2. 服务端 catalog 把 mining_machine 的 `name` 与其他建筑统一口径（查一下为什么只有它是中文）

## 证据

- 实测建造栏 title 列表：`["wind_turbine · 矿 30","matrix_lab · 矿 120 能 60","tesla_tower · 矿 20 能 10","采矿机 · 矿 50 能 20","conveyor_belt_mk1 · 矿 4", ...]`
