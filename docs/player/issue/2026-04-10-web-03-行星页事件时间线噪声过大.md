# 行星页事件时间线默认噪声过大

- 状态：待处理
- 首次记录：2026-04-10
- 来源：`docs/player/已知问题与回归.md` -> `2026-04-10 Web 试玩`
- 类别：Web / 信息设计 / 事件流

## 问题描述

行星页默认事件时间线被大量低价值事件刷屏，关键的建造结果、状态变化和命令失败不容易被玩家第一时间识别。

## 复现步骤

1. 打开 `planet-1-1`
2. 停留数秒并观察默认事件时间线
3. 执行建造或研究等命令后继续观察列表顶部内容

## 实际现象

- 顶部长期出现大量：
  - `回合结算`
  - `资源储量变化`
- 更高价值的：
  - `command_result`
  - `entity_created`
  - `building_state_changed`
  不容易立即看到

## 影响

- 玩家难以判断“刚刚那条操作是否成功”
- 事件面板失去即时反馈与溯源价值
- 失败命令很容易被常规 tick 噪声淹没

## 临时绕过

- 手动筛查更靠后的事件
- 使用 CLI 或直接查事件接口
