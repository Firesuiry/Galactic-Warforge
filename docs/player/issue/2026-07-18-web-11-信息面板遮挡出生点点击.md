# 行星信息面板遮挡出生点区域，挡住对基地附近格子的点击

- 状态：已修复
- 首次记录：2026-07-18
- 来源：2026-07-18 Web 试玩
- 类别：Web / 行星地图 / 交互遮挡

## 问题描述

行星页左上角的信息卡片（PLANET-1-1 / tick / 矿产 / 能量 / 电力 / 研究）以浮层形式压在画布上。基地出生点在地图原点 (3,3) 附近，相机回到原点时，基地、风机、研究站这一片恰好被面板完全遮住——点击基地或矿机时实际点到了面板上，选不中任何对象。

## 实际现象

- 32px/tile 下点击矿机 (5,1)：屏幕坐标落在信息面板区域内，点击被面板拦截，选中状态不变（状态栏仍"未选中对象"）
- 必须先拖拽地图把基地移出面板区域，才能正常点选
- 与 web-10 叠加：玩家开局既要自己找基地，找到了还点不到

## 推荐改进

1. 信息面板改为可折叠，或默认折叠成一条窄 bar
2. 点击命中优先级上让画布穿透该面板（面板区域不阻断 canvas 交互，或面板 pointer-events 只落在真实控件上）
3. 相机回位时自动把基地放在面板遮不住的屏幕位置（如屏幕中心偏右下）

## 证据

- 截图：`docs/player/assets/2026-07-18-web-playtest/50-base-zoom.png`（基地建筑群被面板遮住）、`60-miner-detail.png`（拖开后才能选中）

## 修复记录（2026-07-18，提交：b2742f3）

- 点击穿透此前已由 `.planet-title-chip { pointer-events: none }` 覆盖；本次补齐视觉遮挡与主动避让：
- `PlanetPage.tsx`：信息片新增折叠按钮（`.planet-title-chip__toggle`，单独恢复 `pointer-events: auto`），折叠后只剩"行星名 + tick"窄条；配合 web-10 的回家视角，出生点建筑群居中显示、不再被面板覆盖。
- `styles/index.css`：新增 `.planet-title-chip--collapsed` / `.planet-title-chip__toggle` 样式。
- e2e 同 web-10 用例：断言基地屏幕坐标不落在信息片区域、点击可选中，并验证折叠/展开切换。
