# Web 页面暴露大量开发者视角错误信息

- 状态：待处理
- 首次记录：2026-04-10
- 来源：`docs/player/已知问题与回归.md` -> `2026-04-10 Web 试玩`
- 类别：Web / UX / 错误展示

## 问题描述

当前 Web 页面会直接向玩家展示协议错误、上游代理错误、命令执行细节和 shell 日志，缺乏玩家可理解的产品化错误口径。

## 复现步骤

1. 在 `/agents` 中打开现有失败会话或向新成员发消息
2. 在 `planet` 页执行会失败的研究命令
3. 观察页面中的 system 消息、错误提示和回显文本

## 实际现象

- 页面会直接显示：
  - `action.type is required`
  - `VALIDATION_FAILED`
  - `missing electromagnetic_matrix in research labs`
  - `502 Bad Gateway`
  - shell/codex 调用日志、请求路径、request id

## 影响

- 玩家视角难以理解真实问题
- 大量底层实现细节破坏沉浸感
- 当前前端更像调试工具，而不是玩家客户端

## 临时绕过

- 无产品侧绕过，只能依赖开发者阅读错误详情
