## 修复内容

- 新增 `client-web/src/common/player-facing-error.ts`：`toPlayerFacingMessage` 把协议错误（VALIDATION_FAILED、missing … in research labs）、网关/网络错误（502 Bad Gateway、fetch failed、timeout）、鉴权错误统一翻译成中文玩家口径，无法识别时回退通用文案；已是中文的文案原样放行。原始文本仅保留在 journal 的 `debugMessage` 字段（不渲染）。
- 行星命令流水账（PlanetCommandPanel）：失败条目经 `toPlayerFacingMessage` 渲染；移除 requestId 徽标，改为「等待服务器确认 / 服务器已确认 / 未生效」；去掉文案中的 `/commands authoritative 契约`、`authoritative 命令结果` 等术语。
- `executor.ts` 本地提交失败：展示字段写玩家化文案，原始 error.message 存入 `debugMessage`。
- `planet-commands/error-hints.ts` / `war/error-hints.ts`：删除 detail 里的 `authoritative: <原文>` 前缀，改为中文说明。
- 智能体频道（ChannelWorkspaceView）：失败原因只显示网关公开文案，不再拼 errorCode；删除「真实错误」rawErrorMessage 区块。
- 页面级加载错误（PlanetPage / OverviewPage / WarPage / AgentsPage / LoginPage）统一经 `toPlayerFacingMessage` 渲染。
- `use-war-command.ts`：提交异常与未知回执走玩家化兜底；同时把 `accepted`/`queued` 回执正确识别为成功（顺带修复了既存失败的 Playwright 用例 war-workbench-authoritative，该用例最终断言同步更新为当前 authoritative 行为：登陆可正常发起并显示 landing-1）。
- 测试：`player-facing-error.test.ts` 新增映射用例；executor / AgentsPage / build-workflow / WarPage 相关断言同步更新。

## 问题描述

当前 Web 页面会直接向玩家展示协议错误、上游代理错误、命令执行细节和 shell 日志，缺乏玩家可理解的产品化错误口径。

## 复现步骤

1. 在 `/agents` 中打开现有失败会话或向新成员发消息
2. 在 `planet` 页执行会失败的研究命令
3. 观察页面中的 system 消息、错误提示和回显文本

## 实际现象

- 页面会直接显示：
  - `action.type is required`
  - `VALIDATION_FAILED`
  - `missing electromagnetic_matrix in research labs`
  - `502 Bad Gateway`
  - shell/codex 调用日志、请求路径、request id

## 影响

- 玩家视角难以理解真实问题
- 大量底层实现细节破坏沉浸感
- 当前前端更像调试工具，而不是玩家客户端

## 临时绕过

- 无产品侧绕过，只能依赖开发者阅读错误详情
