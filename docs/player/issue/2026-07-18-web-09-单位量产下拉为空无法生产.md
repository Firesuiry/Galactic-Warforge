# "战斗与制造"单位下拉恒为空，GUI 无法量产任何单位

- 状态：已修复（2026-07-18，提交 hash：b2742f3）
- 首次记录：2026-07-18
- 来源：2026-07-18 Web 试玩
- 类别：Web / 战争 / 单位生产

## 修复内容

- `client-web/src/features/planet-map/PlanetCommandPanel.tsx`：`produceUnitTypes` 过滤条件改为与服务端口径一致的 `production_mode === 'world_produce'`（删除永不命中的 `deploy_command === 'produce'` 分支）。
- 下拉为空时给出提示文案（目录未加载 / 当前没有可量产单位）。
- `client-web/src/pages/PlanetPage.test.tsx`：mock catalog 改为服务端真实返回值 `world_produce`，防止测试再次掩盖口径漂移。
- 新增 Playwright 回归 `client-web/tests/planet-produce-units.spec.ts`：选基地 b-1 → 单位下拉出现 worker/soldier → 下达量产 soldier → 回执受理，已通过真实浏览器验证。

## 问题描述

行星工作台"战斗与制造"页签的"建筑量产"表单里，选了生产建筑（如基地 `b-1`）后，"单位类型"下拉始终为空，"下达量产"按钮无法使用，纯 GUI 无法生产 `worker` / `soldier`。

## 根因

- `PlanetCommandPanel.tsx:271` 过滤条件：`unit.production_mode === 'produce' || unit.deploy_command === 'produce'`
- 服务端 `/catalog` 实际返回 `production_mode = "world_produce"`（`worker`、`soldier` 均是）
- 两边枚举值不一致，过滤结果恒为空

## 影响

- 新局想造兵侦查/防御的玩家在 GUI 里根本找不到入口；战争体验的第一步就断了

## 推荐改进

1. 统一枚举口径：以前端 `shared-client` 的类型定义为准修正服务端返回值，或前端直接兼容 `world_produce`
2. 下拉为空时给出提示（"当前没有可量产单位 / 目录未加载"），而不是一个空白 select 让玩家猜
3. 补一条 e2e：选基地 → 单位下拉出现 soldier → 下达量产 → 回执受理

## 证据

- `/catalog` 实测：`world_units` 仅 2 条，`production_mode` 均为 `world_produce`
- 试玩脚本实测：选中 b-1 后单位下拉 options = `[""]`
