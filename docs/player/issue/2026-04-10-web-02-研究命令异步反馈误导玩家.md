# Web 研究命令的异步反馈误导玩家

- 状态：待处理
- 首次记录：2026-04-10
- 来源：`docs/player/已知问题与回归.md` -> `2026-04-10 Web 试玩`
- 类别：Web / 交互反馈 / 命令系统

## 问题描述

Web 命令面板对异步命令只显示“已接受，将在下一 tick 执行”，不会把 authoritative 执行失败回写到主反馈位，容易误导玩家判断命令已成功。

## 复现步骤

1. 在 Web 行星页完成风机和研究站建造
2. 在未向研究站装矩阵的情况下点击“开始研究 `electromagnetism`”
3. 观察命令面板和事件时间线

## 实际现象

- 命令面板主反馈显示：`accepted, will execute at next tick`
- 真实 authoritative 失败不会覆盖主反馈
- 失败信息只会以事件或快照的形式出现在后续流里

## 影响

- 玩家误以为命令已经成功，只是“稍后生效”
- 玩家很难从当前界面迅速得知失败原因和下一步操作
- 异步命令的反馈链不闭合

## 临时绕过

- 手动查看 `events/snapshot`
- 改用 `client-cli` 读取 `command_result`

## 证据

- Web 面板反馈：`accepted, will execute at next tick`
- authoritative：`evt-6084-1` 返回 `VALIDATION_FAILED: missing electromagnetic_matrix in research labs`
