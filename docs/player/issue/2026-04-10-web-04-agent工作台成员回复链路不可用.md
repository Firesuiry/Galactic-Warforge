# 智能体工作台能创建成员但成员真实回复链路不可用

- 状态：待处理
- 首次记录：2026-04-10
- 来源：`docs/player/已知问题与回归.md` -> `2026-04-10 Web 试玩`
- 类别：Web / Agent 工作台 / 后端集成

## 问题描述

`/agents` 页面可以新建成员、保存权限并发起私聊，但成员无法给出有效回复，最终只会收到系统失败消息。

## 复现步骤

1. 打开 `/agents`
2. 在“成员”页新建成员 `试玩助手`
3. 绑定 `builtin-minimax-api`
4. 保存 `planet-1-1` 与 `observe, build`
5. 发起私聊并发送：`请汇报 planet-1-1 基地附近下一步最适合建造什么，并说明原因。`

## 实际现象

- 成员创建成功，页面显示 `试玩助手 / idle`
- 数秒后收到系统消息：`试玩助手 回复失败：action.type is required`
- 相关会话消息接口也确认只留下玩家消息和一条 system failure

## 影响

- “边玩边创建 agent 做辅助工作”无法真正落地
- 玩家会误以为工作台能力已经可用，但实际任务无法执行

## 临时绕过

- 当前无纯 Web 绕过方案
- 只能排查 `agent-gateway` 的输出 schema / action 结构
