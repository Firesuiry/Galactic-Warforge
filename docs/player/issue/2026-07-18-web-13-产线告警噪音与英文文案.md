## 修复内容（2026-07-18）

说明：根因在 server 的 production_monitor（空配方研究站恒定触发吞吐类告警、AlertMessage 硬编码英文、无同键去重计数），但本次只改客户端展示层；server 侧根治（研究模式跳过、结构化文案、历史计数）留待后续。

1. 新增 `features/production-alerts.ts`：`isResearchStationAlertNoise()` 判定"研究站类建筑（matrix_lab / self_evolution_lab）× 吞吐类告警（throughput_drop/input_shortage/output_blocked/backlog）"为噪音；断电告警（power_shortage/power_low）保留。
2. toast 路径（`event-toasts.ts`）：研究站噪音直接不弹；文案本地化——"风力涡轮机 b-25：原料短缺"（translateBuildingType + translateAlertType），不再使用 server 英文原文 message；mergeKey 加 alert_type 维度（同建筑同原因合并计数）。
3. 展示面同步过滤噪音：行星页活动流告警面板（`PlanetPanels.tsx`）、顶栏告警计数（`TopNav.tsx`）、总览页告警列表（`OverviewPage.tsx`，同时本地化文案）。
4. toast 栈新增"全部忽略"按钮（`NotificationToasts.tsx` + store `dismissAll`）。
5. 测试：`notifications.test.ts` 更新 production_alert 文案/合并键断言，新增研究站噪音过滤与断电保留用例、dismissAll 用例；`PlanetPanels.test.tsx` 新增噪音不进告警面板用例；`TopNav.test.tsx` 新增顶栏计数过滤用例。

## 问题描述

试玩全程右下角持续弹"产线告警 ×N"，内容是英文原文 `building b-25 throughput drop detected`；右侧总览栏反复出现"原料短缺 / 产能下降"警告。触发源是一台空 `matrix_lab`（未配方的研究站）——这是开局的必经状态，却不断刷告警。

## 实际现象

- 同一建筑 b-25 的告警累计到 ×40+，玩家无法判断哪些告警需要处理
- 告警文案未本地化（英文 slug + 建筑内部 ID），玩家看不懂 b-25 是什么
- 空研究站是合法且推荐的玩法状态（文档明确"第一台 matrix_lab 留空当研究站"），不该按"产线掉速"报警

## 推荐改进

1. 告警规则区分建筑角色：研究模式（无配方）的 `matrix_lab` / `self_evolution_lab` 不参与产线吞吐告警
2. 告警文案本地化并带上玩家可读的上下文（"矩阵研究站 b-25：未配置配方，当前作为研究站"）
3. 同一建筑同一原因只保留最新一条并计数，避免刷屏；提供"全部忽略"按钮
