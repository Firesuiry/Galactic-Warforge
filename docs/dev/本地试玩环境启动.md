# 本地试玩环境启动

这份文档只给最短路径：启动服务端、Web 和智能体平台。

如果只想一键拉起整套试玩环境，直接运行：

```bash
cd /home/firesuiry/develop/siliconWorld
bash scripts/start-local-playtest.sh
```

脚本会：

- 清理 `18080` / `18180` / `5173` 旧进程
- 为服务端生成独立试玩数据目录 `.run/local-playtest/server-data/`
- 依次启动 `server`、`agent-gateway`、`client-web`
- 等待健康检查通过
- 把日志和 pid 写到 `.run/local-playtest/`

## 1. 先关旧进程

```bash
kill $(lsof -ti:18080 -ti:18180 -ti:5173) 2>/dev/null || true
```

## 2. 启动服务端

```bash
cd /home/firesuiry/develop/siliconWorld/server
env PATH=/home/firesuiry/sdk/go1.25.0/bin:$PATH \
  go run ./cmd/server -config config-dev.yaml -map-config map.yaml
```

默认地址：

- `http://localhost:18080`

试玩账号：

- `p1 / key_player_1`
- `p2 / key_player_2`

## 3. 启动 Web

```bash
cd /home/firesuiry/develop/siliconWorld/client-web
npm install
npm run dev -- --host 0.0.0.0 --port 5173
```

入口：

- `http://localhost:5173/login`

如果服务端不在 `18080`，改用：

```bash
VITE_SW_PROXY_TARGET=http://localhost:18081 npm run dev -- --host 0.0.0.0 --port 5173
```

## 4. 启动智能体平台

```bash
cd /home/firesuiry/develop/siliconWorld/agent-gateway
npm install
npm run dev
```

默认地址：

- `http://localhost:18180`

Web 里的智能体工作台通过 `/agent-api` 代理到这个端口。

如果仓库根目录 `.env` 里已有 MiniMax key，启动后会自动注入一个内置模型 Provider `builtin-minimax-api`，可直接在智能体工作台里绑定给成员。

## 5. 最小检查

```bash
curl http://localhost:18080/health
curl http://localhost:18180/health
curl http://localhost:5173/agent-api/health
```

预期：

- 三条都返回 `{"status":"ok"...}` 或 `{"status":"ok"}`

如果 `http://localhost:5173/agent-api/health` 失败，通常不是前端问题，而是 `agent-gateway` 没启动。
