# server 现状与差距

更新时间：2026-04-05

## 1. 一句话结论

当前 `server/` 已经不是“只有概念的空框架”，而是一个能跑通中前期工业闭环、带基础战争，并且已经具备一条官方可复现的戴森中后期、终局能源/胜利态，以及高阶单位与同星系舰队最小闭环的服务端游戏核心；但它仍然不是《戴森球计划》的完整后期体验，也还没有打通多星球长期经营闭环。

## 2. 当前版本更像什么

和原版 DSP 相比，当前实现更接近一个：

- 多人
- 阵营对抗
- API / 命令驱动
- 2D 平面地图
- 以 `active planet` 为操作焦点、但允许多个已加载行星并行保存 runtime
- 工业生产 + 防御 / 战争

的服务器化工业战争游戏。

## 3. 已经稳定存在的主循环

当前已经形成的核心闭环大致是：

- 研究 -> 解锁 -> 建造
- 发电 -> 电网 -> 建筑运行
- 采矿 -> 物流 / 仓储 -> 冶炼 / 生产
- 基础防御 -> 敌袭 -> 战斗结算
- 官方 midgame 场景下的 `轨道采集 -> 太阳帆 / 戴森脚手架 -> 火箭发射 -> 射线接收收益反馈`
- 官方 midgame 场景下的 `高阶单位科研 -> recipe 生产 -> transfer_item -> deploy_squad|commission_fleet -> runtime 查询 / fleet_attack`

已有调研文档显示，主循环内已经串起施工、科研、发电、电网、射线接收、太阳帆、戴森球、采矿、传送带、分拣器、建筑 IO、管线、生产周期、仓储、物流调度、战斗结算，以及终局燃料态 / 胜利态与空间舰队运行态。

## 4. 与 DSP 的关键差异

- 不是机甲视角，而是命令驱动的多人服务器。
- 星球是 2D 网格，不是球面铺设。
- 当前命令焦点仍围绕 `active planet`，但已加载行星的 runtime / networks 与最小星际物流闭环已经接通。
- 战斗和敌对势力比原版 DSP 权重更高。
- 对外接口天然面向 CLI、Web 和 AI，而不是只服务单一前端。

## 5. 仍然明显存在的缺口

当前最重要的缺口不是“完全没写”，而是“很多内容已经存在 authoritative runtime，但玩家入口仍偏受控验证”。重点包括：

- 多星球完整殖民仍未做完，但“已加载行星 + active planet 切换 + 同星系星际物流”的最小经营闭环已经可用。
- 戴森中后期与终局高阶单位虽然已经有官方可验证路线，但更像“受控验证闭环”，不是普通新局自然主线。
- `build_dyson_*` 这类戴森结构入口仍偏实验性脚手架，而不是完整材料化规划系统。
- 高阶单位线已经公开，但 `fleet_attack` 目前仍只覆盖同一恒星系目标，尚未扩展到跨星系远征、返航补给、玩家对玩家太空战等更复杂玩法。
- 完整 DSP 式后期体验仍未形成。
