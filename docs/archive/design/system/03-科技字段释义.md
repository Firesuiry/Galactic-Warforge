# 科技字段释义（对应配置规范 V1）

## 1. 说明范围
- 本文解释 [02-科技配置规范.md](/home/firesuiry/develop/siliconWorld/docs/archive/design/system/02-科技配置规范.md) 中“科技完整字段”的语义。
- 只解释字段含义，不给具体科技的最终数值。
- 字段命名兼容《Dyson Sphere Program》Research 页面常见展示项。

## 2. 科技完整字段释义

### 2.1 基础字段
- `spec_version`：配置规范版本号；当前固定为 `v1`。
- `kind`：对象类型；科技必须为 `tech`。
- `id`：科技唯一标识；全局唯一，且需与文件名一致。
- `name`：科技显示名称（用于 UI、日志、事件文本）。
- `name_en`：英文名称（多语言或外部资料对齐用途）。
- `category`：一级分类（如 `main`、`logistics`、`energy`、`combat`、`mecha`）。
- `subcategory`：二级分类（如 `oil`、`power_grid`、`missile`）。
- `tech_tier`：科技层级标记，用于树形布局、排序与筛选。

### 2.2 `ui`（界面展示）
- `ui.icon`：科技图标资源标识。
- `ui.position.x`：科技在科技树画布中的横坐标。
- `ui.position.y`：科技在科技树画布中的纵坐标。

### 2.3 `research`（研究成本与模式）
- `research.data_volume_hash`：研究哈希需求（对应 DSP `Data Volume`）。
- `research.items_needed`：研究所需物品清单（对应 DSP `Items Needed`）。
- `research.items_needed[].item_id`：消耗物品 ID。
- `research.items_needed[].count`：该物品总需求数量。
- `research.items_per_set`：每组研究消耗件数（对应 DSP `Items/Set`，用于研究站节拍与吞吐估算）。
- `research.mode`：研究模式。
- `single`：一次性科技，完成后不可重复。
- `repeatable`：可重复科技，按等级持续研究。

### 2.4 `research.repeatable`（重复研究参数）
- `research.repeatable.max_level`：重复上限。
- `0`：非重复科技（仅当 `mode=single`）。
- `-1`：无限重复。
- `>0`：最多重复指定次数。
- `research.repeatable.data_volume_scale`：每提升一级时 `data_volume_hash` 的倍率。
- `research.repeatable.item_cost_scale`：每提升一级时 `items_needed` 的倍率。

### 2.5 `prerequisites`（前置条件）
- `prerequisites.required_techs`：显式前置科技 ID 列表（对应 DSP `Prereqs`）。
- `prerequisites.hidden_requirements`：隐藏前置条件列表（用于隐藏科技/特殊触发）。
- `prerequisites.hidden_requirements[].type`：隐藏条件类型（如 `item_obtain`）。
- `prerequisites.hidden_requirements[].target_id`：条件目标 ID（由 `type` 决定是物品/事件/目标对象）。
- `prerequisites.hidden_requirements[].count`：条件所需数量。

### 2.6 `unlocks`（解锁内容）
- `unlocks.buildings`：解锁建筑 ID 列表。
- `unlocks.recipes`：解锁配方 ID 列表。
- `unlocks.abilities`：解锁系统能力/规则开关 ID 列表。
- `unlocks.units`：解锁单位 ID 列表。

### 2.7 `leads_to`（后继关系）
- `leads_to`：可选显式后继科技 ID 列表（对应 DSP `Leads To`）。
- 说明：该字段可由 `required_techs` 反向推导；保留它可减少前端构图查询开销。

### 2.8 `visibility`（可见性）
- `visibility.hidden`：是否为隐藏科技。
- `visibility.show_in_tree`：是否在科技树中展示节点（可用于“已触发后才显示”的策略）。

### 2.9 `metadata`（元信息）
- `metadata.source`：来源标签（如 `dsp_reference`、`original_design`）。
- `metadata.tags`：标签数组，用于筛选、统计、运营活动绑定或调试。

## 3. DSP 字段对照速查
- `Data Volume` -> `research.data_volume_hash`
- `Items Needed` -> `research.items_needed[]`
- `Items/Set` -> `research.items_per_set`
- `Prereqs` -> `prerequisites.required_techs[]`
- `Unlocks` -> `unlocks.*`
- `Leads To` -> `leads_to[]`
- 隐藏前置（如特殊掉落）-> `prerequisites.hidden_requirements[]`

## 4. 字段使用建议
- 科技关系以 `required_techs` 为唯一主真相，`leads_to` 仅做冗余缓存。
- 重复科技要明确增长模型，避免后续版本出现“成本回退”或“无法收敛”的数值问题。
- 隐藏前置建议结构化表达，不要把业务规则写成自然语言字符串。
- `unlocks` 推荐拆分到具体维度（建筑/配方/单位/能力），避免使用一个通用数组混放不同类型目标。
- 运行态数据（玩家研究进度、队列、当前等级）不得进入静态配置。
