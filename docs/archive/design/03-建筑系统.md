# 建筑系统设计

## 1. 设计目标
- 建立统一的建筑定义目录，支持可验证、可扩展、可热替换。
- 通过“分类 + 子类 + 占地 + 造价 + 规则”描述建筑能力。
- 支撑建造、升级、拆除、运行时参数与生产挂载。

## 2. 建筑定义模型
- 核心字段：
  - `id`：建筑类型唯一标识
  - `name`：建筑名称
  - `category/subcategory`：分类与子类
  - `footprint`：占地宽高
  - `build_cost`：矿物、能量、物品造价
  - `unlock_tech`：解锁科技
  - `buildable`：是否允许建造
  - `requires_resource_node`：是否必须贴资源点
  - `can_produce_units`：是否可生产单位

## 3. 建筑分类与功能

### 3.1 采集类（collect）
- `mining_machine`：矿脉采集入口。
- `advanced_mining_machine`：高效采矿。
- `water_pump`：液体资源抽取，必须贴 `water` 资源点，运行时带本地存储与输出口。
- `oil_extractor`：油井抽取，必须贴 `crude_oil` 资源点，运行时带本地存储与输出口。
- `orbital_collector`：气态行星轨道采集。

### 3.2 运输类（transport）
- `conveyor_belt_mk1/mk2/mk3`：不同吞吐档位的传送带。
- `splitter`：分流与汇流。
- `automatic_piler`：堆叠提升带宽利用率。
- `traffic_monitor`：流量监测与告警触发。
- `spray_coater`：喷涂增产/加速效果。
- `sorter_mk1/mk2/mk3`、`pile_sorter`：建筑与传送带间搬运。

### 3.3 仓储类（storage）
- `depot_mk1/depot_mk2`：基础与高容量仓储。
- `storage_tank`：液体仓储。

### 3.4 生产与精炼类（production/refining/chemical）
- 冶炼：`arc_smelter`、`plane_smelter`、`negentropy_smelter`
- 制造：`assembling_machine_mk1/mk2/mk3`、`recomposing_assembler`
- 化工：`chemical_plant`、`quantum_chemical_plant`
- 专项：`oil_refinery`、`fractionator`、`miniature_particle_collider`

### 3.5 科研类（research）
- `matrix_lab`：矩阵生产与研究。
- `self_evolution_lab`：高阶研究设施。

### 3.6 物流枢纽类（logistics_hub）
- `logistics_distributor`：配送分发。
- `planetary_logistics_station`：行星内物流枢纽。
- `interstellar_logistics_station`：跨星球物流枢纽。

### 3.7 电力与电网类（power/power_grid）
- 发电：`wind_turbine`、`thermal_power_plant`、`solar_panel`、`geothermal_power_station`、`mini_fusion_power_plant`、`artificial_star`
- 输配电：`tesla_tower`、`wireless_power_tower`、`satellite_substation`
- 储放电：`energy_exchanger`、`accumulator`、`accumulator_full`

### 3.8 戴森相关类（dyson）
- `em_rail_ejector`：通过输入口装载 `solar_sail` 后发射太阳帆。
- `vertical_launching_silo`：发射火箭建造结构。
- `ray_receiver`：接收轨道能量或产出临界光子。

### 3.9 指挥防御类（command_signal）
- 防御：`gauss_turret`、`missile_turret`、`implosion_cannon`、`laser_turret`、`plasma_turret`、`sr_plasma_turret`
- 支援：`battlefield_analysis_base`、`jammer_tower`、`signal_tower`
- 行星防护：`planetary_shield_generator`
- 地形工具：`foundation`

## 4. 建造/升级/拆除规则
- 建造：
  - 校验地块可建、占地不冲突、资源点约束、造价可支付、科技已解锁。
  - `mining_machine` / `water_pump` / `oil_extractor` 属于资源点采集器，缺少对应资源点时直接拒绝建造。
- 升级：
  - 受升级规则约束（最大等级、倍率、持续 Tick）。
- 拆除：
  - 受返还率和持续 Tick 约束，处理中间态占格恢复。

## 5. 运行时状态
- 每个建筑实例保留：
  - 所属玩家、位置、朝向、等级、运行参数、I/O 缓冲。
- 与系统耦合：
  - 物流系统读取输入输出口与缓存。
  - 电网系统读取功耗/发电属性。
  - 生产系统读取配方与周期参数。
