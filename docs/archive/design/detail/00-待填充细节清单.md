# 设计细节待填充清单（基于 DSP 对标）

## 1. 文档目的
- 本清单用于把 `docs/design` 现有“框架级设计”补齐到“可直接实现”的粒度。
- 细节来源以《戴森球计划（Dyson Sphere Program）》机制为主，优先参考：
  - https://dyson-sphere-program.fandom.com/wiki/Buildings
  - https://dyson-sphere-program.fandom.com/wiki/Category%3ABuildings
- 本清单当前只列“需要补什么”，不在本文件内直接给出最终数值和公式常量。

## 2. 通用补全标准（所有分册都要补）
- [ ] 明确静态配置字段：类型、单位、默认值、最小/最大值、是否可热更新。
- [ ] 明确运行时状态字段：生命周期、状态机、状态迁移条件。
- [ ] 明确 Tick 结算顺序：先后依赖、读写边界、跨系统同步点。
- [ ] 明确命令契约：输入参数、前置校验、失败码、幂等规则。
- [ ] 明确事件契约：触发条件、事件负载、可见性裁剪规则。
- [ ] 明确异常策略：断电、断链、库存满、资源不足、并发冲突。
- [ ] 明确平衡参数来源：DSP 对标值、后端可调系数、难度倍率。
- [ ] 明确测试用例：单元测试、集成测试、回放一致性测试。

## 3. 按分册的待填充细节

### 3.1 `01-世界与地图系统.md`
- [ ] 行星地形枚举与语义：可建设、不可建设、水域、岩浆、特殊地形。
- [ ] 资源节点模型：矿脉/油井/水域/气态资源的节点字段、容量、再生/枯竭规则。
- [ ] 行星类型参数：普通行星、气态行星、特殊行星的建造与采集限制。
- [ ] 生成参数：资源分布算法、种子复现、稀有资源出现条件。
- [ ] 路径与邻接规则：平面网格寻路、建筑占地形状与旋转规则。
- [ ] 视野底图切片：迷雾缓存粒度、增量更新策略。

### 3.2 `02-玩家与执行体系统.md`
- [ ] 玩家资源模型拆分：原始资源池 vs 物品库存 vs 能量预算。
- [ ] 执行体任务系统：建造/升级/拆除/操作任务的排队、抢占、取消规则。
- [ ] 执行体可达判定：范围、地形阻挡、跨区域操作限制。
- [ ] 多执行体扩展接口：协同、冲突解决、任务分片策略。
- [ ] 权限模型细化：命令级权限、队伍共享权限、战争状态临时权限。

### 3.3 `03-建筑系统.md`
- [ ] 建筑定义 schema：占地、朝向、端口、功耗/发电、输入输出缓存。
- [ ] 建造规则：贴矿/贴油/贴水/轨道专属等特殊落位约束。
- [ ] 升级链规则：Mk 级继承、参数增量、升级中状态与回滚规则。
- [ ] 拆除返还规则：返还比例、是否保留内部缓存、拆除耗时。
- [ ] 建筑运行状态机：运行、待机、缺料、堵塞、断电、停机。
- [ ] 建筑与蓝图：批量放置、批量升级、镜像/旋转一致性约束。

### 3.4 `04-物流与生产系统.md`
- [ ] 传送带段模型：带段容量、带速、方向、合并与分流优先级。
- [ ] 分拣器模型：抓取节拍、距离衰减、堆叠搬运规则。
- [ ] 物流站模型：槽位配置、供需模式切换、最小保留量、阈值触发。
- [ ] 无人机/运输舰调度：任务分配、装载策略、充能策略、路径选择。
- [ ] 配方模型：输入输出比、周期、功耗、喷涂增益/加速增益。
- [ ] 堵塞传播规则：上游停滞、旁路优先、异常恢复策略。

### 3.5 `05-能源与电网系统.md`
- [ ] 发电建筑参数：基础发电、波动性、燃料类型、燃料热值映射。
- [ ] 输配电覆盖：供电半径、连接规则、跨网合并/分裂重算。
- [ ] 储能策略：充放电阈值、效率损耗、优先级策略。
- [ ] 负载优先级：基础物流、防御、科研、终局产线的分级策略。
- [ ] 射线接收与电网耦合：发电模式与产光子模式切换规则。

### 3.6 `06-科技系统.md`
- [ ] 科技树结构：主干科技、前置关系、层级、重复研究项。
- [ ] 矩阵消耗模型：不同矩阵的研究点转换率与并行研究限制。
- [ ] 解锁效果映射：建筑解锁、配方解锁、全局倍率、单位能力提升。
- [ ] 研究设施模式：矩阵生产模式与研究模式切换成本。
- [ ] 科技事件：开始、暂停、完成、前置失效等事件定义。

### 3.7 `07-戴森球系统.md`
- [ ] 戴森蜂群模型：太阳帆轨道、寿命、衰减与补帆逻辑。
- [ ] 结构建造模型：节点/框架/壳层的配方、建造进度与应力消耗。
- [ ] 火箭与发射井：发射队列、发射条件、轨道窗口约束。
- [ ] 产能结算：蜂群产能与结构产能叠加规则、损耗与转换效率。
- [ ] 临界光子链路：接收站切换临界光子模式的条件与产率公式。

### 3.8 `08-战斗与防御系统.md`
- [ ] 敌方模型：黑雾单位类型、抗性、行为树、波次与威胁等级。
- [ ] 炮塔模型：射程、攻击间隔、伤害类型、目标优先级、弹药需求。
- [ ] 护盾模型：容量、回复、受击衰减、过载保护。
- [ ] 侦测模型：信号塔/分析基站的探测半径与信息精度等级。
- [ ] 战斗结算顺序：侦测 -> 锁定 -> 开火 -> 伤害 -> 状态更新 -> 事件。

### 3.9 `09-命令事件与可见性系统.md`
- [ ] 命令分类：建造、物流配置、配方切换、研究、战斗指令、戴森编辑。
- [ ] 幂等与重放：重复命令识别、去重键、重放一致性边界。
- [ ] 事件字典：每个系统的标准事件类型、必填字段、兼容策略。
- [ ] 可见性裁剪：敌我视野、盟友共享、侦测级别差异化下发。
- [ ] 审计与回滚：快照频率、命令日志保留策略、回滚粒度。

## 5. 每个建筑的“必填细节字段模板”
- [ ] 基础信息：`id/name/category/subcategory/tech_tier`。
- [ ] 建造约束：占地、朝向、可建地形、是否贴资源点、是否仅气态/轨道放置。
- [ ] 解锁条件：前置科技、前置建筑、前置物品。
- [ ] 建造成本：物品清单、建造 Tick、拆除返还。
- [ ] 功能模式：工作模式列表与模式切换条件。
- [ ] I/O 端口：输入输出端口类型、带宽、缓存容量、优先级。
- [ ] 功率模型：待机功耗、工作功耗、峰值功耗、发电功率或转换效率。
- [ ] 速率模型：采集速率/加工速率/搬运速率/发射速率。
- [ ] 消耗品模型：燃料、弹药、喷涂剂、维护品等消耗规则。
- [ ] 运行状态机：运行、待机、堵塞、缺料、缺电、故障等状态及迁移。
- [ ] 事件输出：开停机、缺料、堵塞、过载、完成、受击等事件。
- [ ] 升级链：可升级目标、升级耗时、属性继承与覆盖策略。
- [ ] 战斗属性（如适用）：血量、护盾、抗性、攻击参数。
- [ ] 可见性属性：侦测半径、被侦测等级、战争迷雾下可见规则。
- [ ] 测试用例：构造、运行、断电、堵塞、升级、拆除、重放一致性。

## 6. 后续实现建议的切分顺序
- [ ] 第一批（P0）：采集、传送带、分拣器、冶炼、基础发电、基础电网、基础仓储。
- [ ] 第二批（P1）：物流站、无人机/运输舰、喷涂、科研、行星防御闭环。
- [ ] 第三批（P2）：戴森发射链、射线接收与临界光子、终局发电与终局制造。
