# 世界与地图系统设计

## 1. 设计目标
- 在服务端维护权威世界状态，支持星系、恒星系、行星三级查询。
- 行星内使用二维平面地图承载建造、采集、战斗与物流。
- 为战争迷雾与侦测系统提供可裁剪的地形与实体信息。

## 2. 地图数据结构
- 行星基础：
  - `planet_id`
  - `map_width`
  - `map_height`
  - `terrain[y][x]`
- 网格格子字段：
  - 坐标：`x, y`
  - 地形类型：可建设、阻挡、水域、岩浆等
  - 占用信息：建筑占用、资源节点占用

## 3. 平面地图规则
- 边界规则：所有坐标必须满足 `0 <= x < map_width` 且 `0 <= y < map_height`。
- 占格规则：同一格不能被多个互斥实体重复占用。
- 建造规则：需校验地形可建性、占地矩形完整性、冲突与资源点约束。
- 距离规则：默认采用曼哈顿距离用于执行体操作范围校验。

## 4. 行星生成与初始化
- 初始化流程：
  - 生成地形矩阵与环境参数。
  - 生成资源分布（矿脉、油井、水域、特殊资源）。
  - 构建运行态网格并注入地形。
- 目标：同配置输入下具备可复现性，便于测试与回放对账。

## 5. 对外查询能力
- 世界摘要：返回 Tick、玩家概要、关键资源与告警状态。
- 星系/恒星系查询：返回层级结构与进度概览。
- 行星查询：返回地图尺寸、地形、资源与建筑概况。
- 迷雾查询：返回玩家可见区域与侦测结果。

## 6. 与其他系统耦合点
- 建筑系统：提供占格与地形可建判断。
- 物流系统：提供路径拓扑与站点落位坐标。
- 战斗系统：提供探测半径与射程判定基础。
- 可见性系统：作为实体裁剪和事件过滤底图。
