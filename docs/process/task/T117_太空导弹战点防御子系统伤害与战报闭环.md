# T117 太空导弹战、点防御、子系统伤害与战报闭环

## 背景

当前太空舰队战斗仍偏简化，更多是“舰队对目标造成固定伤害”的模型，缺少：

- 导弹 / 鱼雷
- 点防御
- 电子压制下的命中变化
- 子系统损伤
- 可解释战报

设计文档明确要求高阶太空战形成：

- 导弹与点防的对抗
- 护盾 / 装甲 / 结构 / 子系统分层受损
- 齐射窗口
- 凝聚度与撤退
- battle report

这是把现有最小舰队线升级为真正太空战争系统的核心任务。

## 需求细节

1. 太空战必须支持导弹 / 鱼雷 / 点防。
2. 伤害必须分层落到：
   - 护盾
   - 装甲
   - 结构
   - 子系统
3. 战斗结果必须能生成战报，而不是只能通过最终 HP 推断。

## 改动需求

1. 扩展太空战武器体系，至少区分：
   - 直射武器
   - 导弹 / 鱼雷
   - 点防御
   - 电子压制武器
2. 引入分层伤害结算：
   - 先护盾
   - 再装甲
   - 再结构
   - 最后对子系统造成失效或降效
3. 至少支持以下子系统受损后果：
   - 引擎失效或降速
   - 火控失效导致命中下降
   - 传感器失效导致锁定恶化
   - 点防失效导致拦截能力崩塌
4. 点防与导弹必须形成真实对抗：
   - 导弹不是必中
   - 点防不是全拦
   - 干扰与诱饵会改变结果
5. 生成 battle report，至少包含：
   - 参战任务群 / 舰队
   - 火力构成
   - 拦截情况
   - 子系统受损
   - 战损与撤退结果
6. 新增必要事件：
   - `missile_salvo_fired`
   - `point_defense_intercept`
   - `battle_report_generated`

## 不在本任务范围

- 不要求完成登陆作战
- 不要求完成 Web 战报面板
- 不要求完成 AI 舰队参谋

## 验收标准

1. 太空战不再只是固定伤害交换，而是存在导弹、点防和子系统伤害。
2. 电子战和情报质量会真实影响太空战结果。
3. battle report 可稳定查询，不需要读源码才能解释战斗过程。
4. 保存 / 读档 / 回放后，太空战事件与战报保持一致。
