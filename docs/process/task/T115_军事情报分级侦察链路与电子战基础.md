# T115 军事情报分级、侦察链路与电子战基础

## 背景

当前战争迷雾主要还是“可见 / 不可见”的二元状态，不足以支撑：

- 侦察收益
- 电子战
- 隐形
- 虚假目标
- 远程导弹战的锁定质量

设计文档要求敌方目标至少分成：

1. `unknown_signal`
2. `classified_contact`
3. `confirmed_type`
4. `fully_resolved`

如果没有这层情报分级，后续太空导弹战、制轨权和 AI 情报官都只能建立在过于粗糙的可见性之上。

## 设计参考

- `docs/dev/战争组件设计.md`
  - 第 10 章“侦察、迷雾与电子战”
  - 第 7.4 节中的信号 / 隐形约束

## 需求细节

1. 敌方目标必须从“实体可见性”提升为“接触情报对象”。
2. 情报质量必须受以下因素影响：
   - 传感器类型
   - 距离
   - 目标信号特征
   - 主动 / 被动探测
   - 电子干扰
3. 电子战要能影响：
   - 接触分级
   - 锁定稳定度
   - 导弹制导质量
   - 虚假目标生成

## 改动需求

1. 引入 authoritative `contact` / `sensor_contact` 模型。
2. 至少支持以下情报等级：
   - 未知信号
   - 已分类接触
   - 已确认类型
   - 完整解析
3. 传感器来源至少覆盖：
   - 视野
   - 主动雷达
   - 被动电磁监听
   - 红外
   - 信号塔 / 指挥设施
   - 侦察单位
4. 电子战至少覆盖：
   - 干扰敌方锁定
   - 降低情报精度
   - 诱导导弹偏航
   - 伪造虚假目标
5. Query 层至少能查询：
   - 某星球 / 恒星系当前接触列表
   - 每个接触的情报等级
   - 来源传感器
   - 最近更新时间

## 不在本任务范围

- 不要求完成完整导弹战
- 不要求完成 AI 情报官
- 不要求完成 Web 情报面板

## 验收标准

1. 敌方目标不再只是简单“可见 / 不可见”，而是具备分级接触对象。
2. 主动探测、被动探测、距离、干扰会真实改变接触分级与稳定度。
3. 电子战可在 runtime 中影响接触质量或锁定质量，而不是只提供文案字段。
4. 查询面可稳定返回接触列表及其等级变化。
