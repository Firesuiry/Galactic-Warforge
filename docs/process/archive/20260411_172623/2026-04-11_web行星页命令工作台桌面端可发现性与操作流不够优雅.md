# 2026-04-11 Web 行星页命令工作台桌面端可发现性与操作流不够优雅

## 背景

- 试玩日期：2026-04-11
- 试玩方式：真实浏览器进入：
  - 新局：`http://127.0.0.1:4173/planet/planet-1-1`
  - midgame：`http://127.0.0.1:4174/planet/planet-1-2`
- 已确认当前页面功能本身大体可用：
  - 地图可显示
  - 图层切换可用
  - 命令面板能提交建造/装料/发射/射线模式切换
  - authoritative 结果会回写到右侧工作台

问题更多出在“信息与操作怎么被组织给玩家”。

## 主要问题

### 1. 首屏右栏优先显示状态摘要，关键可操作表单隐藏得太深

桌面端首次进入行星页时，右栏默认先看到的是：

- 当前路由行星
- active planet
- 最近命令结果
- 工作流标签按钮

但真正可操作的 typed form 大多在右栏内部更深处，需要继续在内部滚动后才能看到。

结果是：

- 首次进入页面时，不熟悉的人会误以为右栏只是“状态面板”
- 很难第一眼发现“这里其实能直接建造/装料/戴森发射”
- 对“主要用 Web 游玩”的目标不够友好

### 2. 地图、事件、告警、工作台同时竞争首屏注意力，操作主线不够聚焦

当前桌面端布局同时把以下内容放在同一视觉层级：

- 地图
- 图层开关
- 工作台摘要
- 事件时间线
- 告警面板

这对调试很有帮助，但对真正推进玩法并不够聚焦。

尤其在新局阶段，玩家更需要的是：

- 我下一步应该做什么
- 我能点哪里
- 为什么这条命令失败了

而不是让“活动流”和“告警”抢占与操作区相近的视觉权重。

### 3. 失败反馈是准确的，但前置成本信息仍然不够前摄

本轮在新局 Web 中通过建造表单尝试继续建 `wind_turbine` 时，页面正确返回：

- `need 30 minerals, have 20`

说明 authoritative 错误回写是正常的。

但在点击前，建造区没有把当前关键成本约束前摄到足够醒目的位置，例如：

- 当前矿产剩余
- 目标建筑造价
- 是否可支付

所以玩家往往还是要“点一次失败”才知道当前不够造。

## 影响

- 功能虽然能用，但页面更像“调试控制台”，不够像“主游玩界面”
- 新玩家或回归测试者需要额外摸索，才能发现真正的操作入口
- 桌面端的操作效率不如能力本身成熟度

## 改动需求

1. 重排桌面端首屏优先级：
   - 让当前工作流的核心表单更早进入首屏
   - 降低“结果摘要 + 活动流 + 告警”对首屏操作空间的挤占
2. 强化工作流可发现性：
   - 切换到某个工作流后，首个核心表单应直接可见
   - 不应要求用户先意识到右栏内部还需要二次滚动
3. 对建造/装料等高频动作增加前摄信息：
   - 当前资源
   - 当前建筑成本
   - 是否满足提交条件
4. 给桌面端增加真实浏览器回归：
   - 首屏能直接看到当前工作流的主操作表单
   - 不额外内部滚动即可完成最小建造或装料动作

## 验收标准

- 桌面端首次进入 `/planet/:planetId` 时，玩家能立即看见当前工作流的核心表单
- “地图观察”和“命令推进”不再互相抢焦点
- 建造等高频动作在点击前就能理解关键资源约束
- 页面整体更接近“主游玩客户端”，而不是“只适合研发自测的调试台”
