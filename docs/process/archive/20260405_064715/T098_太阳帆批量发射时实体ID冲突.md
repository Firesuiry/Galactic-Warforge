## 当前问题：一次批量发射多个太阳帆时，所有帆共用了同一个 `entity_id`

### 复现

1. 启动官方 midgame 场景，切到 `planet-1-1`，补一套可运行电网，并建出：
   - `build 8 3 em_rail_ejector`
   - `build 9 3 vertical_launching_silo`
   - `build 10 3 ray_receiver`
2. 通过戴森脚手架保证接收站与发射建筑进入可运行状态：
   - `build_dyson_node sys-1 0 10 20 --orbit-radius 1.2`
   - `build_dyson_node sys-1 0 -10 -20 --orbit-radius 1.2`
   - `build_dyson_frame sys-1 0 p1-node-l0-latp1000-lonp2000 p1-node-l0-latm1000-lonm2000`
   - `build_dyson_shell sys-1 0 -15 15 0.4`
3. 给电磁轨道弹射器装填太阳帆：
   - `transfer b-39 solar_sail 4`
4. 一次性发射 4 个太阳帆：
   - `launch_solar_sail b-39 --count 4 --orbit-radius 1.2`
5. 观察 CLI SSE 事件流中的 `entity_created`

### 实际现象

- `launch_solar_sail` 命令本身返回成功：
  - `OK: launched 4 solar sail(s) into orbit`
- 但同一 tick 的 4 条 `entity_created` 事件都给出了同一个太阳帆实体 ID：
  - `entity_id = sail-p1-5038`
  - 同一 ID 连续出现 4 次
- 这意味着：
  - 本轮发射出来的 4 个太阳帆并没有拿到 4 个稳定可区分的实体标识
  - 如果后续有依赖 `entity_id` 的跟踪、销毁、审计、回放或可视化，就无法可靠地区分每一张帆

### 进一步研判

- 这是根据当前实现作出的直接结论：
  - `server/internal/gamecore/solar_sail_settlement.go` 中的 `LaunchSolarSail(...)` 目前把太阳帆 ID 生成为：
    - `sail-<playerID>-<launchTick>`
  - 当一次命令在同一个 tick 内发射多个太阳帆时，所有帆都会拿到同一个 ID
- 这不是设定差异，而是实体标识设计错误。

### 影响

- SSE / event snapshot / audit 无法稳定表达“同一次批量发射中的多张独立太阳帆”。
- 后续如果要做：
  - 太阳帆生命周期跟踪
  - 单帆销毁事件
  - 回放校验
  - Web 可视化展示
  - AI 基于事件流做精确计数
  都会出现歧义或覆盖风险。
- 当前问题虽然不阻断“总能量增加”，但会破坏事件和实体层的一致性，因此必须收口。

### 改动要求

- 太阳帆实体 ID 必须改成真正唯一，而不是只依赖 `launchTick`。
- 至少保证同一个玩家在同一个 tick 内批量发射多张太阳帆时：
  - 每一张帆都有不同的 `entity_id`
  - `entity_created` 事件中的 `entity_id` 与 orbit 内部保存的实体 ID 一一对应
  - 后续 `entity_destroyed` 等事件也能准确对应到单帆
- 不能通过“把一次批量发射压缩成一个伪实体”来规避问题；如果系统模型里保存的是多张独立帆，就必须对外暴露可区分的独立 ID。
- 增加回归测试，至少覆盖：
  - 同 tick `launch_solar_sail --count 4` 时，orbit 内 `Sails` 的 4 个 `ID` 全部唯一
  - 同 tick 发出的 4 条 `entity_created` 事件，其 `entity_id` 全部唯一
  - 后续寿命结算销毁时，不会因为 ID 冲突导致事件重复或丢失

## 验收标准

1. 一次命令发射 4 个太阳帆后，事件流里必须出现 4 个不同的 `entity_id`。
2. orbit 运行态里保存的 4 张太阳帆也必须分别具有唯一 ID。
3. 太阳帆过期销毁时，`entity_destroyed` 必须能和当初创建的唯一 ID 一一对应。
4. 修复后不能回退当前已走通的能力：
   - `launch_solar_sail`
   - `ray_receiver`
   - `build_dyson_*`
   - `launch_rocket`
