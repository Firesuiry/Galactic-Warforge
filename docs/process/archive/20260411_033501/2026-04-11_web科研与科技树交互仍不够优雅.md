# 2026-04-11 Web 科研与科技树交互仍不够优雅

## 背景

- 试玩日期：2026-04-11
- 试玩方式：`client-web` 为主，`client-cli` 辅助核对
- 结论：默认新局现在已经可以只靠 Web 走通最小科研起步链

本轮已确认的成功路径：

1. `planet-1-1` 建 `wind_turbine`
2. 建空 `matrix_lab`
3. 通过 Web `研究与装料` 页把 `10 electromagnetic_matrix` 装入研究站
4. 在同页启动 `electromagnetism`
5. authoritative 返回 `electromagnetism 研究完成`

## 当前问题

虽然功能闭环已通，但科研 UI 仍然不够优雅，也不利于玩家按阶段推进。

### 1. 科技入口仍是扁平大下拉

当前 Web 把整棵公开科技树直接放进一个扁平下拉中，从开局第一屏就能看到：

- `electromagnetism`
- `solar_sail_orbit`
- `vertical_launching`
- `annihilation`
- `mission_complete`

等全部层级节点。

缺少这些关键信息：

- 哪些是当前可达
- 哪些已完成
- 哪些还被前置锁住
- 每项需要什么矩阵
- 会解锁哪些建筑 / 配方

结果是：

- 功能“能点”，但引导“不自然”
- 开局玩家会在同一控件里同时看到终局科技
- 很难快速理解当前应该优先研究什么

### 2. 命令反馈文案仍有明显过时口径

在 midgame Web 里给 `em_rail_ejector` 装 `solar_sail`、给 `vertical_launching_silo` 装 `small_carrier_rocket` 后，成功提示仍会显示：

- `物料已装入，下一步可继续启动 dyson_sphere_program。`

这在当前上下文里明显不对：

- midgame 已经远超 `dyson_sphere_program`
- 装入太阳帆 / 火箭后的下一步应分别指向：
  - 发射太阳帆
  - 建戴森脚手架
  - 发射火箭
  - 切换射线接收站模式

## 影响

- 默认新局虽然不再被功能 blocker 卡住，但研究体验仍偏“调试面板”
- 科技树缺少阶段化表达，不符合“主要用 Web 游玩”的预期
- 成功反馈缺少上下文感，削弱了中后期玩法指引

## 改动需求

1. 把科研入口从扁平下拉改成阶段化视图，至少区分：
   - 当前可研究
   - 已完成
   - 尚未满足前置
2. 每个科技节点都应显示最少必要信息：
   - 前置科技
   - 所需矩阵
   - 解锁建筑 / 配方
3. 默认新局首屏应显式提示推荐路径：
   - `风机 -> 空研究站 -> 装 10 电磁矩阵 -> 研究 electromagnetism`
4. 中后期装料成功文案要改成上下文相关提示，至少区分：
   - 研究站装料
   - 电磁弹射器装料
   - 发射井装料
   - 射线接收站模式切换后的下一步
5. 增加浏览器回归：
   - 默认新局纯 Web 完成 `electromagnetism`
   - midgame 装入 `solar_sail` / `small_carrier_rocket` 后提示文案正确

## 验收标准

- 玩家在 Web 中能直接看懂“当前阶段能研究什么、为什么能研究、研究后会得到什么”
- 开局与 midgame 的成功反馈不再复用错误或过时的文案
- 默认新局与 midgame 的科研 / 装料面板都能给出明确、上下文正确的下一步指引
