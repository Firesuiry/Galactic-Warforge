# T093 戴森终局科技树与能源局势统计缺口

## 任务结论

- 2026-04-04，T093 已按设计目标完成收口。
- 本轮没有继续扩成“完整太空舰队系统”，而是坚持去伪存真：能在当前 authoritative world + query + command 架构里闭环的内容直接落地，超出当前承载层的高阶单位线明确降级。

## 已落地结果

### 1. 终局弹药与科技解锁闭环

- `antimatter_capsule`、`gravity_missile` 已补齐到 runtime `item` / `recipe`。
- 两条配方都通过 `recomposing_assembler` 进入真实生产闭环，继续复用公开 `build <x> <y> recomposing_assembler --recipe <recipe_id>` 入口。
- `/catalog.items`、`/catalog.recipes`、`/catalog.techs[].unlocks` 现已与运行时一致：
  - `mass_energy_storage -> recipe: antimatter_capsule`
  - `gravity_missile -> recipe: gravity_missile`
- 本轮没有顺手改成“炮塔必须吞终局弹药”；当前收口范围是科技、物品、配方、catalog 与产线闭环，不是整套供弹语义重写。

### 2. 高阶 unit unlock 去伪存真

- `normalizeTechUnlocks()` 已把 `TechUnlockUnit` 也纳入 runtime 校验。
- 当前真正保留为 runtime 支持的 unit unlock 只有已有实际承载层的物流单位。
- `engine` 仍保留为前置科技，但不再对外宣称可解锁玩家可玩的高阶单位。
- `prototype`、`precision_drone`、`corvette`、`destroyer` 已统一 `hidden=true`，因为当前版本仍没有公开生产、部署、编队、轨道/星系级查询与太空战结算入口。

### 3. `mission_complete -> game_win` 已接入真实胜利逻辑

- `victory_rule` 已从死配置变成运行时语义，支持：
  - `elimination`
  - `mission_complete`
  - `hybrid`
- 仓库当前提供的 `config.yaml`、`config-dev.yaml`、`config-midgame.yaml` 已统一改为 `battlefield.victory_rule = hybrid`。
- `mission_complete` 完成后会立即触发 `game_win`，并生成完整胜利态：
  - `/state/summary` 返回 `winner` / `victory_reason` / `victory_rule`
  - `GET /events/snapshot` 可见 `victory_declared`
  - `GET /audit` 可见 `action = victory`
  - `save.json.runtime_state` 持久化 `winner` / `victory_reason` / `victory_rule` / `victory_tech_id`
  - 回放与回滚会恢复同一份 victory state
- `research_completed` 事件继续保留；若 `mission_complete` 在该 tick 完成，事件顺序为先 `research_completed`，再 `victory_declared`。

### 4. 能源摘要统计已改为真实结算值

- `energy_stats.generation` 改为聚合当前 active planet 上玩家所属 power network 的真实 `supply`。
- `energy_stats.consumption` 改为聚合 power allocation 的真实 `allocated`。
- `energy_stats.current_stored` 改为读取各储能建筑的 `energy_storage.energy`。
- `energy_stats.shortage_ticks` 只在本 tick 任一玩家网络 `shortage=true` 时累加。
- 结果是以下几类查询已使用同一套 runtime 事实来源：
  - `/world/planets/{planet_id}/networks`
  - `/state/summary.players[pid].stats.energy_stats`
  - `/state/stats.energy_stats`
  - `inspect building` 中的储能状态

## 当前边界

- 高阶舰队线仍未开放给玩家：当前没有 `prototype`、`precision_drone`、`corvette`、`destroyer` 的公开生产/部署/战斗玩法。
- `build_dyson_node` / `build_dyson_frame` / `build_dyson_shell` 仍是实验性脚手架入口，主要负责科技校验与结构写入，不额外扣建筑材料。
- `summary` / `stats` 仍保持 active planet 语义，不做跨所有已加载行星的总汇总。

## 验收口径

- `mass_energy_storage` 与 `gravity_missile` 不再出现“科技存在但 unlock 丢失”。
- `mission_complete` 不再只是装饰性科技，而是真正会宣告 `game_win`。
- `summary/stats` 的能源统计不再复用静态建筑参数或建筑 HP。
- 高阶 unit unlock 不再以“当前可玩内容”误导玩家。
