## 当前仍未实现的问题

### 问题 1：终局高阶舰队线仍未对玩家开放，不能继续宣称 DSP 相关科技树与玩法已经“都实现”

#### 复现

1. 在官方 midgame 局查询目录：
   - `curl -sf -H 'Authorization: Bearer key_player_1' http://127.0.0.1:18152/catalog | jq '{hidden_techs:[.techs[]|select(.hidden==true)|.id], public_units:[.units[]|.id]}'`
2. 在 CLI 执行：
   - `help produce`
   - `produce u-2 corvette`
   - `produce b-1 corvette`
   - `produce b-1 soldier`

#### 实际现象

- `/catalog.techs` 里以下 4 项仍然是 `hidden=true`：
  - `prototype`
  - `precision_drone`
  - `corvette`
  - `destroyer`
- `/catalog.units` 当前仍只公开：
  - `worker`
  - `soldier`
- `help produce` 当前只显示：
  - `Produce a server-public world unit from /catalog.units: worker, soldier`
- CLI 实测表明当前玩家侧并没有可用的终局舰队入口：
  - `produce u-2 corvette` 返回 `ENTITY_NOT_FOUND: building u-2 not found`
  - `produce b-1 corvette` 返回 `INVALID_TARGET: can only produce units at a production building`
  - `produce b-1 soldier` 同样返回 `INVALID_TARGET: can only produce units at a production building`
- 也就是说，当前不仅没有 `corvette / destroyer` 的公开生产能力，连公开的单位生产入口本身也已经不再符合旧复现口径；玩家侧仍缺：
  - 高阶单位生产入口
  - 高阶单位部署 / 编队入口
  - 轨道 / 星系级单位查询入口
  - 对应战斗结算与玩家可观察结果

#### 影响

- 当前版本虽然已经覆盖了大量 DSP 中后期建筑和戴森主链，但终局高阶舰队线仍然没有真正开放给玩家。
- 这不属于本仓库明确列出的 4 条设计差异，因此仍然应视为“未实现的 DSP 相关玩法”，而不是可忽略差异。
- 因此，当前不能把仓库状态描述成“戴森球计划相关建筑、科技树、玩法都已经实现”。
- 现有文档中的旧复现口径 `produce u-2 corvette -> unit corvette is not publicly producible` 也已经过时，会误导后续回归与试玩。

## 改动要求

### 1. 补齐终局高阶舰队线的公开玩法闭环

- 目标科技 / 单位：
  - `prototype`
  - `precision_drone`
  - `corvette`
  - `destroyer`
- 必须补齐至少以下内容：
  - 玩家可见科技树显示
  - 公开单位目录
  - 合法的生产入口
  - 部署 / 编队入口
  - 轨道 / 星系级查询入口
  - 对应的战斗结算与玩家可观察结果

### 2. 统一 `produce` 的玩家入口语义

- 现在 `produce` 的第一个参数已经按“建筑 ID”解释，旧文档和旧问题记录仍在按“单位 ID”复现。
- 必须统一以下内容：
  - `client-cli help produce`
  - `docs/dev/客户端CLI.md`
  - `docs/dev/服务端API.md`
  - `docs/player/已知问题与回归.md`
- 至少要让玩家明确知道：
  - `produce` 作用于什么目标
  - 哪些建筑能生产哪些公开单位
  - 高阶舰队线当前是否已真正开放

### 3. 若短期仍不能开放高阶舰队线，必须显式维持“未实现”口径

- 在真正补齐玩法前，所有玩家侧文档都必须明确写成：
  - 这条线当前仍未实现 / 未开放
  - 不能再使用任何“已经完整覆盖 DSP 科技树 / 终局玩法”的措辞
- 但这条文档收口不能代替实现本身；该问题仍应保留在任务列表，直到玩家真的能玩到这条线。

## 验收标准

1. `/catalog.techs` 中 `prototype`、`precision_drone`、`corvette`、`destroyer` 不再只是隐藏名词，玩家可以通过公开科技树看到并推进这条线。
2. `/catalog.units` 与 CLI / API 中存在与这条线一致的公开入口，不再只剩 `worker` / `soldier`。
3. 玩家能通过公开命令真实完成至少一条高阶单位链路：
   - 研究
   - 生产
   - 部署 / 查询
   - 看到实际战斗或可观察运行结果
4. `produce` 的目标语义、帮助文本、错误口径与文档完全一致，不再出现旧文档仍让玩家执行 `produce u-2 corvette` 的情况。
5. 本轮已确认可用的中后期建筑与戴森主链不能回退。
