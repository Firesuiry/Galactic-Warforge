# T119 行星层机甲战、前线据点与轨道支援闭环

## 背景

战争设计文档要求行星层承担：

- 机甲、地面载具、无人机
- 基地火力网
- 前线推进
- 据点争夺
- 登陆后的滩头阵地扩张
- 轨道支援与反轨道压力

当前仓库虽然已有地面单位和敌对势力基础，但离“可玩的行星层战争”还差：

- 机甲底盘与蓝图化
- 前线据点
- 防空 / 防轨
- 轨道支援联动
- 地面任务群持续推进

## 设计参考

- `docs/dev/战争组件设计.md`
  - 第 5.1 节“行星层”
  - 第 6 章“单位与组织体系”

## 需求细节

1. 地面战争单位必须从当前简化模型扩展到机甲 / 地面载具 / 无人机任务群。
2. 行星层必须存在前线据点和滩头阵地概念。
3. 轨道支援必须能正向影响地面战，而不是完全分离。

## 改动需求

1. 让行星层支持蓝图化机甲 / 载具 / 无人机部署，而不是只停留在固定地面单位。
2. 增加前线据点 / 滩头阵地 / 防空阵地等关键战术对象。
3. 地面任务群至少支持：
   - 占点
   - 推进
   - 守点
   - 清障
   - 伴随补给
4. 明确轨道支援联动：
   - 有制轨优势时可提供轨道打击或火力支援
   - 行星护盾、防空、防轨设施可对抗轨道支援
5. Query 层至少可查询：
   - 当前前线据点归属
   - 滩头阵地状态
   - 地面任务群推进状态
   - 轨道支援冷却 / 可用性

## 不在本任务范围

- 不要求完成 Web 地面战争可视化美术
- 不要求完成 AI 地面指挥官
- 不要求新增完整剧情战役脚本

## 验收标准

1. 玩家可以把蓝图化地面战争单位部署到行星层，并组织成任务群推进。
2. 前线据点 / 滩头阵地是 authoritative 对象，可被占领、争夺、摧毁或巩固。
3. 轨道支援与行星防御存在真实对抗，不是两个互不相干的系统。
4. 行星层战局能被制轨权与登陆链路真实改变。
