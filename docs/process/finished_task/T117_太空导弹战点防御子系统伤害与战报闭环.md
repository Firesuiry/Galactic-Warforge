# T117 太空导弹战、点防御、子系统伤害与战报闭环

## 背景

当前太空舰队战斗仍偏简化，更多是“舰队对目标造成固定伤害”的模型，缺少：

- 导弹 / 鱼雷
- 点防御
- 电子压制下的命中变化
- 子系统损伤
- 可解释战报

设计文档明确要求高阶太空战形成：

- 导弹与点防的对抗
- 护盾 / 装甲 / 结构 / 子系统分层受损
- 齐射窗口
- 凝聚度与撤退
- battle report

这是把现有最小舰队线升级为真正太空战争系统的核心任务。

## 需求细节

1. 太空战必须支持导弹 / 鱼雷 / 点防。
2. 伤害必须分层落到：
   - 护盾
   - 装甲
   - 结构
   - 子系统
3. 战斗结果必须能生成战报，而不是只能通过最终 HP 推断。

## 改动需求

1. 扩展太空战武器体系，至少区分：
   - 直射武器
   - 导弹 / 鱼雷
   - 点防御
   - 电子压制武器
2. 引入分层伤害结算：
   - 先护盾
   - 再装甲
   - 再结构
   - 最后对子系统造成失效或降效
3. 至少支持以下子系统受损后果：
   - 引擎失效或降速
   - 火控失效导致命中下降
   - 传感器失效导致锁定恶化
   - 点防失效导致拦截能力崩塌
4. 点防与导弹必须形成真实对抗：
   - 导弹不是必中
   - 点防不是全拦
   - 干扰与诱饵会改变结果
5. 生成 battle report，至少包含：
   - 参战任务群 / 舰队
   - 火力构成
   - 拦截情况
   - 子系统受损
   - 战损与撤退结果
6. 新增必要事件：
   - `missile_salvo_fired`
   - `point_defense_intercept`
   - `battle_report_generated`

## 不在本任务范围

- 不要求完成登陆作战
- 不要求完成 Web 战报面板
- 不要求完成 AI 舰队参谋

## 验收标准

1. 太空战不再只是固定伤害交换，而是存在导弹、点防和子系统伤害。
2. 电子战和情报质量会真实影响太空战结果。
3. battle report 可稳定查询，不需要读源码才能解释战斗过程。
4. 保存 / 读档 / 回放后，太空战事件与战报保持一致。

## 完成情况

- 完成时间：2026-04-20
- 结果：已完成
- 实现摘要：
  - 新增 authoritative 太空战模型：`SpaceWeaponMix`、`DurabilityLayerState`、`SpaceFleetSubsystemState`、`SpaceBattleReport`，把舰队火力构成、装甲/结构耐久层、子系统状态和战报正式纳入 `SpaceFleet` / `SpaceRuntimeState`。
  - `rebuildFleetStats` 现在会基于蓝图组件目录聚合直射火力、导弹、点防和电子战强度，并初始化舰队的护盾、装甲、结构与子系统状态。
  - `settleSpaceFleets` 已从“固定伤害打 enemy_force”升级为：
    - 基于 `contacts.lock_quality` / `jamming_penalty` / `missile_drift_risk` 结算命中和导弹漂移；
    - 区分直射火力、导弹齐射与点防拦截；
    - 按护盾 -> 装甲 -> 结构 -> 子系统的顺序结算伤害；
    - 在子系统层真实体现 `engine` / `fire_control` / `sensors` / `point_defense` 的降效或失效；
    - 生成并持久化 `battle_reports`，同时触发 `missile_salvo_fired` / `point_defense_intercept` / `battle_report_generated` 事件。
  - Query 与类型口径已同步：
    - `/world/systems/{system_id}/runtime` 新增 `battle_reports`
    - `fleets[]` / `/world/fleets*` 新增 `weapons` / `armor` / `structure` / `subsystems` / `last_battle_report(_id)`
    - `shared-client/src/types.ts` 与 `docs/dev/服务端API.md` 已同步上述字段
  - snapshot/save/load 已通过 `CloneSpaceRuntimeState` 覆盖新增太空战状态与战报，确保回放和恢复可稳定读取。
- 边界说明：
  - 本任务仍然基于当前仓库现有的 `fleet_attack -> enemy_force` 主链路结算太空战，不额外引入完整的敌方舰队 runtime 或 Web 战报工作台。
  - 当前 battle report 聚焦 authoritative 结算与查询闭环；更复杂的多舰队战区协同、登陆与 UI 呈现留给后续任务。
