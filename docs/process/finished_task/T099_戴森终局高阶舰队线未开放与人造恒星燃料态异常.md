# T099 戴森终局高阶舰队线未开放与人造恒星燃料态异常

## 问题背景

- 2026-04-04 对当前工作区代码做了两套隔离实测：
  - 默认新局：`18141`
  - 官方 midgame：`18142`
- 本轮目标不是重复确认 4 月 4 日白天已经收口的中后期链路，而是继续核对“当前版本是否已经把 DSP 相关建筑、科技树、玩法全部实现”。
- 以下明确设计差异不计入缺陷范围：
  - 上帝视角 + 执行体，而不是原版机甲直操
  - 多人 / 阵营对抗服务端
  - API 驱动、无渲染
  - 行星为 2D 平面网格

## 当前仍未实现 / 新发现的问题

### 问题 1：终局高阶舰队线仍未对玩家开放，DSP 终局科技树和玩法不能算“全部实现”

#### 复现

1. 在官方 midgame 局查询 `/catalog`，观察：
   - `prototype`
   - `precision_drone`
   - `corvette`
   - `destroyer`
2. 在 CLI 执行：
   - `produce u-2 corvette`
   - `help produce`

#### 实际现象

- `/catalog.techs` 中上述 4 项仍然是 `hidden=true`
- CLI 实测返回：
  - `unit_type 必须是 worker 或 soldier`
- `help produce` 只暴露：
  - `Produce unit (worker/soldier)`
- 当前没有公开的：
  - 高阶单位生产入口
  - 高阶单位部署/编队入口
  - 轨道/太空单位查询入口
  - 对应战斗结算与玩家可观察结果

#### 影响

- 当前版本虽然已经覆盖了大量 DSP 中后期建筑和戴森主链，但终局高阶舰队线仍然缺失。
- 因此不能继续把当前实现描述成“DSP 相关建筑、科技树、玩法已经全部实现”。
- 这会直接误导后续试玩结论、能力盘点和玩家预期。

### 问题 2：`artificial_star` 在没有燃料的情况下仍显示 `running`

#### 复现

1. 使用官方 midgame 场景启动新局并登录 `p1`
2. 在 `planet-1-2` 执行：
   - `build 11 5 artificial_star`
3. 查询：
   - `inspect planet-1-2 building b-73`

#### 实际现象

- `b-73.runtime.functions.energy.fuel_rules` 明确要求：
  - `antimatter_fuel_rod`
- 但当前官方 midgame 玩家背包没有 `antimatter_fuel_rod`
- `b-73` 自身本地存储里也没有燃料
- 在这种情况下，`inspect` 仍返回：
  - `runtime.state = running`

#### 影响

- 玩家、Web/CLI 展示层以及自动化脚本会把“可建但未供燃料”的终局电站误判成已正常工作。
- 即使后续结算层没有真正放出满额供电，当前运行态口径也已经和玩法语义不一致。
- 这会污染终局电力闭环验证与后续调试判断。

## 改动要求

### 1. 对终局高阶舰队线做明确收口，不能继续保持“隐藏但仍被算作已覆盖”

- 目标科技 / 玩法线：
  - `prototype`
  - `precision_drone`
  - `corvette`
  - `destroyer`
- 必须二选一并彻底收口：
  - 方案 A：真正实现这条公开玩法线
    - 补齐科技公开显示
    - 补齐玩家可用的生产 / 部署 / 编队 / 查询 / 战斗入口
    - 补齐事件、查询和文档
  - 方案 B：继续隐藏，但同步把所有“当前版本已完整覆盖 DSP 科技树 / 终局玩法”的表述改成不成立
- 无论采用哪种方案，都必须让玩家文档、CLI 文档、服务端 API 文档和能力盘点文档口径一致。

### 2. 修正 `artificial_star` 的燃料门禁与运行态

- 在没有 `antimatter_fuel_rod` 时：
  - 不能继续显示 `running`
  - 不能把该建筑误判为已正常工作
- 需要明确统一以下观察面：
  - `inspect`
  - `scene`
  - `building_state_changed`
  - `state/stats.energy_stats`
  - `world/planets/{planet_id}/networks`
- 有燃料时再进入正常运行，并体现为真实供电收益。

### 3. 补验证场景与自动化测试

- 至少增加一套能验证终局能源建筑与终局玩法边界的自动化测试或专用验证场景：
  - `artificial_star` 空燃料 -> 非 `running`
  - `artificial_star` 装入燃料 -> `running` 且供电上升
  - 高阶舰队线若仍隐藏，测试必须锁定：
    - `/catalog.techs[].hidden`
    - 玩家侧命令面不宣称可用
    - 文档不再宣称“已全部实现”

## 验收标准

1. 对终局高阶舰队线，仓库口径必须统一为以下二者之一：
   - 玩家已经能通过公开科技树 + 公开命令真实体验该玩法
   - 当前仍未实现，且所有玩家侧文档都明确写清，不再对外宣称“已全部实现”
2. `artificial_star` 在无燃料时，`inspect` / `scene` / SSE 不能再显示为 `running`。
3. 当 `artificial_star` 装入 `antimatter_fuel_rod` 后，玩家能从查询面看到明确的运行态变化与供电收益变化。
4. 本轮已确认可用的中后期建筑和戴森命令链路不能回退。
