# T112 战争蓝图校验状态机与预置改型闭环

## 背景

`T111` 只解决了底盘、组件和公开蓝图目录的 authoritative 数据结构问题。下一步必须把“蓝图不只是数据表条目，而是可创建、可校验、可定型、可派生改型的对象”真正打通。

`docs/dev/战争组件设计.md` 已明确要求：

- 功率约束
- 体积约束
- 质量约束
- 刚性约束
- 热负荷约束
- 信号 / 隐形约束
- 维护约束
- 蓝图状态机：`draft -> validated -> prototype -> field_tested -> adopted -> obsolete`

如果这一步不先落地，后续生产、部署和补给都只能继续依赖静态单位表。

## 设计参考

- `docs/dev/战争组件设计.md`
  - 第 7.4 节“强制约束”
  - 第 7.5 节“蓝图状态机”
  - 第 7.6 节“改型系统”

## 需求细节

1. 玩家必须能拥有自己的战争蓝图对象。
2. 蓝图至少要支持：
   - 创建草案
   - 安装 / 替换组件
   - 校验是否合法
   - 定型
   - 基于父蓝图创建改型
3. 校验失败不能只返回布尔值，必须返回结构化原因。
4. 改型只能在父蓝图基础上做受控修改，不能变成“另起一套无约束新蓝图”。

## 改动需求

1. 为蓝图建立完整状态机，并明确每个状态允许的操作：
   - `draft`
   - `validated`
   - `prototype`
   - `field_tested`
   - `adopted`
   - `obsolete`
2. 至少实现以下 authoritative 能力：
   - 创建蓝图
   - 修改槽位组件
   - 校验蓝图
   - 定型蓝图
   - 从父蓝图派生改型
   - 查询蓝图详情 / 校验结果
3. 校验结果必须覆盖并区分以下约束：
   - 功率
   - 体积
   - 质量
   - 刚性
   - 热负荷
   - 信号 / 隐形
   - 维护成本
4. 对非法蓝图，服务端需要返回稳定、可解释、可测试的结构化错误，例如：
   - `power_budget_exceeded`
   - `volume_budget_exceeded`
   - `hardpoint_mismatch`
   - `rigidity_budget_exceeded`
   - `heat_dissipation_insufficient`
5. 预置蓝图与玩家蓝图必须走同一套校验语义，不能出现：
   - 预置蓝图绕过约束
   - 玩家蓝图比预置蓝图多一套额外规则
6. 改型要明确：
   - 继承父蓝图的底盘
   - 允许修改哪些组件槽位
   - 如何记录父子关系
   - 如何避免父蓝图被子蓝图反向污染

## 不在本任务范围

- 不要求打通军工生产线
- 不要求打通 task force / theater
- 不要求做 CLI / Web 交互表单

## 验收标准

1. 玩家可通过 authoritative 接口创建、编辑、校验、定型蓝图。
2. 非法蓝图会返回结构化失败原因，而不是模糊报错。
3. 蓝图状态机会真实约束可执行动作，不能在 `obsolete` 或未通过校验的状态下直接量产。
4. 预置蓝图与玩家蓝图共用同一套约束与校验逻辑。
5. 改型具备明确父子关系，并且不会退化成无约束复制。

### 本轮落地

- 新增 authoritative `WarBlueprintDefinition`、预算/热/信号/维护校验、线性状态机与受控改型规则，玩家蓝图对象挂到 `PlayerState` 并进入 snapshot clone。
- 新增蓝图命令链路：`blueprint_create`、`blueprint_set_component`、`blueprint_validate`、`blueprint_finalize`、`blueprint_variant`、`blueprint_set_status`，统一走 `/commands`。
- `blueprint_validate` 失败不再只给布尔结论，`results[].details.validation` 和 `GET /war/blueprints/{blueprint_id}` 会返回结构化 issue 列表与预算使用量。
- 新增 `GET /war/blueprints` 与 `GET /war/blueprints/{blueprint_id}` 查询面；只返回当前玩家私有蓝图，不混入公开预置目录。
- 新增 `blueprint_validated` / `blueprint_invalidated` 事件；已验证蓝图被再次编辑时会自动打回 `draft` 并清空最近一次校验。
- `shared-client` 已同步命令枚举、`CommandResult.details`、战争蓝图类型与查询 helper；`docs/dev/服务端API.md` 已更新。
