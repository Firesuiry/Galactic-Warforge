# T089 官方戴森中后期验证场景闭环

## 问题背景

- 2026-04-03 使用 `client-cli` 对当前工作区代码做了两轮隔离实测：
  - `config-midgame.yaml + map-midgame.yaml`，端口 `18094`
  - 在其基础上额外预解锁 `solar_sail_orbit` / `ray_receiver` 的临时场景，端口 `18095`
- 已确认当前实现里以下内容已经可用，不属于本任务范围：
  - `/catalog` 已存在并正确暴露 `titanium_crystal`、`titanium_alloy`、`frame_material`、`quantum_chip`、`small_carrier_rocket`
  - `vertical_launching` 已解锁 `vertical_launching_silo` 与 `small_carrier_rocket`
  - `solar_sail_orbit` 已解锁 `em_rail_ejector` 与 `solar_sail`
  - `orbital_collector` 可在气态行星建造并实际产出 `hydrogen` / `deuterium`
  - `build_dyson_node` / `build_dyson_frame` / `build_dyson_shell` / `demolish_dyson` 命令可用
  - `ray_receiver` 在存在戴森结构时可实际产出 `critical_photon`
- 当前真正未闭环的是：**官方中后期验证场景仍无法把“太阳帆发射 / 火箭发射”作为玩家可执行路线走通。**

## 复现问题 1：官方 midgame 路线缺少最小供电闭环

### 复现步骤

1. 启动 `config-midgame.yaml + map-midgame.yaml`
2. 登录 `p1`
3. 在基地附近执行：
   - `build 3 1 orbital_collector`
   - `build 4 1 vertical_launching_silo`

### 实际现象

- 建筑都能成功建出
- 但创建事件里的运行态立刻变为：
  - `orbital_collector -> no_power / power_out_of_range`
  - `vertical_launching_silo -> no_power / power_out_of_range`
- 必须额外补 `tesla_tower + 多台 wind_turbine`，它们才会进入 `running`

### 影响

- `docs/player/上手与验证.md` 4.2 当前“先建 collector / 再建 silo”的官方 quick route 缺少关键供电步骤
- 玩家即使按文档执行，也无法直接进入后续的载荷装填与发射验证

## 复现问题 2：官方 midgame 场景没有玩家可达的载荷装填路径

### 复现 A：垂直发射井

1. 在官方 midgame 场景中补齐供电，让 `vertical_launching_silo` 进入 `running`
2. 执行 `build_dyson_node sys-1 0 10 20 --orbit-radius 1.2`
3. 执行 `launch_rocket <silo_id> sys-1 --layer 0`

### 实际现象 A

- `launch_rocket` 返回：
  - `INSUFFICIENT_RESOURCE: need 1 small_carrier_rocket loaded, have 0`
- `vertical_launching_silo` 虽然默认配方已挂 `small_carrier_rocket`
- 但建筑长期表现为 `running + input_shortage`
- `production_monitor.total_moves = 0`
- 官方 midgame 给了玩家背包物资 `frame_material / deuterium_fuel_rod / quantum_chip`
- 但当前没有玩家可见命令能把这些背包物品装进发射井本地存储，也没有预置供料链

### 复现 B：电磁发射器

1. 在额外预解锁 `solar_sail_orbit` 的隔离场景中建好并供电 `em_rail_ejector`
2. 玩家背包中额外预置 `solar_sail x16`
3. 直接执行 `launch_solar_sail <ejector_id>`

### 实际现象 B

- `launch_solar_sail` 返回：
  - `INSUFFICIENT_RESOURCE: need 1 solar sails loaded, have 0`
- `em_rail_ejector` 本地 `storage` 为空
- 玩家背包中的 `solar_sail` 不会自动成为发射器载荷
- 当前 CLI / API 里也没有显式的“背包 -> 建筑本地存储”装填入口

### 影响

- 官方中后期场景当前只能验证：
  - 气态行星入口
  - 戴森脚手架命令
  - 射线接收结算
- 但无法把两条关键发射链路真正作为玩家玩法回归项跑通：
  - `launch_solar_sail`
  - `launch_rocket`
- `docs/player/上手与验证.md` 与 `docs/player/玩法指南.md` 中关于官方 midgame 快速验证路线的发射步骤，当前都无法按文档直接完成

## 改动要求

### 目标

- 把官方 `config-midgame.yaml + map-midgame.yaml` 场景补成**真正可回归的戴森中后期验证场景**

### 必须满足的改动方向

- 需要提供一条明确、玩家可执行、无需隐藏注入的载荷装填路径，至少满足以下方案之一：
  - 方案 A，推荐：官方 midgame 直接预置可运行的供电与供料条件，让 `launch_solar_sail`、`launch_rocket` 都能在少量命令内成功
  - 方案 B：新增玩家可见的“背包物品装填到建筑本地存储”命令/API，并同步 CLI / 文档
  - 方案 C：保留现有 API，但 midgame 场景必须预放完整且可达的自动供料链路，确保发射器 / 发射井能拿到载荷
- 无论采用哪种方案，都要同步修正文档中的官方 midgame 路线，写清楚：
  - 供电步骤
  - 供料步骤
  - 成功发射的最短命令路径

## 需要同步更新的文档

- `docs/player/上手与验证.md`
- `docs/player/玩法指南.md`
- 若引入新 API / CLI：
  - `docs/dev/服务端API.md`
  - `docs/dev/客户端CLI.md`

## 验收标准

1. 使用仓库内官方 `config-midgame.yaml + map-midgame.yaml` 启动新局
2. 登录后只按文档提供的公开命令推进，不允许手改存档或隐藏注入
3. 能稳定完成以下回归：
   - `launch_solar_sail` 返回 `OK`
   - `launch_rocket` 返回 `OK`
   - `event_snapshot` 中出现 `rocket_launched`
4. 文档中的 midgame 快速验证路线能被新的真实试玩逐步复现
