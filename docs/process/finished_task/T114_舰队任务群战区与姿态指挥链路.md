# T114 舰队、任务群、战区与姿态指挥链路

## 背景

当前战争运行态更接近“单个 squad / fleet 接收少数直接命令”，缺少：

- 编制模板
- 任务群
- 战区
- 指挥带宽
- 战斗姿态

`docs/dev/战争组件设计.md` 已明确战争不应只靠单体单位直控，而要支持：

- `task_force`
- `theater`
- `command_capacity`
- `hold / patrol / escort / intercept / harass / siege / bombard / retreat_on_losses`

如果不先把指挥链路建起来，后面的补给、情报、电子战、登陆和 AI 自治都没有稳定挂点。

## 设计参考

- `docs/dev/战争组件设计.md`
  - 第 6 章“单位与组织体系”
  - 第 9 章“编制、姿态与战区”

## 需求细节

1. 允许玩家把单位组织成任务群，而不是只对单个 squad / fleet 发命令。
2. 引入战区对象，表达：
   - 主战区
   - 次战区
   - 禁入区
   - 集结区
   - 补给优先区
3. 引入 `command_capacity`，对超编行为给出真实惩罚。
4. 姿态必须成为 authoritative 运行态的一部分，而不是 UI 文案。

## 改动需求

1. 建立任务群 / 舰队 / 战区的数据模型和 authoritative runtime。
2. 至少支持以下命令或等价能力：
   - 创建任务群
   - 分配成员
   - 设置姿态
   - 部署到目标位置 / 目标星球 / 目标恒星系
   - 设置战区目标
3. 姿态必须真实影响运行态：
   - 目标优先级
   - 交战距离
   - 是否追击
   - 是否优先保持隐形
   - 是否达到损失阈值后撤退
4. 引入 `command_capacity`，至少要能表达：
   - 来源：指挥中心 / 指挥舰 / 战场分析基站 / 军事 AI 核心
   - 消耗：任务群规模、成员复杂度、跨层指挥负担
   - 超编惩罚：延迟、命中下降、队形维持变差、协同衰减
5. Query 层至少要能返回：
   - 当前任务群成员
   - 当前姿态
   - 所属战区
   - 当前指挥容量与占用

## 验收标准

1. 玩家可创建任务群 / 战区，并把 runtime 单位编入任务群。
2. 姿态会真实影响运行态行为，而不是只写在配置里。
3. `command_capacity` 存在 authoritative 数据来源、消耗口径和超编惩罚。
4. 查询面可稳定观察任务群、战区、姿态和指挥容量。

## 完成情况

- 已新增 authoritative 战争组织层状态 `WarCoordination`，挂载到 `PlayerState` 下，覆盖：
  - `task_forces`
  - `theaters`
- 已新增任务群与战区命令链路：
  - `task_force_create`
  - `task_force_assign`
  - `task_force_set_stance`
  - `task_force_deploy`
  - `theater_create`
  - `theater_define_zone`
  - `theater_set_objective`
- 已把姿态接入运行态结算：
  - squad / fleet 会按任务群姿态影响目标选择
  - `hold` / `intercept` / `retreat_on_losses` 已真实影响交战距离、追击与撤退
  - `harass` 等姿态会通过 profile 影响隐形偏好与伤害折损
- 已新增 `command_capacity` authoritative 计算与惩罚：
  - 来源覆盖 `command_center` / `command_ship` / `battlefield_analysis_base` / `military_ai_core`
  - 消耗覆盖任务群成员规模与复杂度
  - 超编会真实反映为 `delay_penalty` / `hit_penalty` / `formation_penalty` / `coordination_penalty`
- 已补齐查询与对外接口：
  - `GET /world/warfare/task-forces`
  - `GET /world/warfare/theaters`
  - `shared-client/src/types.ts`
  - `shared-client/src/api.ts`
  - `docs/dev/服务端API.md`
- `task_force_deploy` 当前实现的是 authoritative 部署意图与战区绑定，不伪装成已完成的自动跨星系航渡系统。
