# T032 建筑升级与拆除规则落地

## 需求细节
- 建筑升级链：升级条件、耗材、耗时与替换规则。
- 拆除与回收：回收率、返还物品与特殊限制。
- 与建筑定义与运行参数联动，确保升级/拆除对状态与库存的影响明确。
- 规则实现保持直接、干净，不引入兼容层。

## 实现结果

- 升级/拆除的 `require_idle` 规则改为严格要求建筑处于 `idle` 状态。
- 补充升级与拆除的物品消耗/返还测试用例。
