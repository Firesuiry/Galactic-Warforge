# T092 默认新局科研起点锁死，无法从开局合法进入 DSP 主线

## 问题背景

- 2026-04-03 对当前工作区做了两套隔离实测：
  - 默认新局：`config-dev.yaml + map.yaml` 派生配置，端口 `18121`
  - 官方 midgame：`config-midgame.yaml + map-midgame.yaml` 派生配置，端口 `18122`
- 本任务只关心“当前项目宣称可从普通新局一路推进到中期工业化”的说法是否真实成立。
- 以下明确设计差异不计入缺陷范围：
  - 上帝视角 + 执行体，而不是原版机甲直操
  - 多人 / 阵营对抗服务端
  - API 驱动、无渲染
  - 行星为 2D 平面网格

## 本轮已确认可用的部分

- 官方 midgame 的戴森中后期公开链路大体可用，不应再按旧问题重复记录为缺失：
  - `switch_active_planet planet-1-1` / 切回 `planet-1-2` 返回 `OK`
  - `build 6 2 orbital_collector` 在补足电网后进入 `running`
  - `inspect planet-1-2 building b-40` 显示其库存已累计：
    - `hydrogen = 1000`
    - `deuterium = 826`
  - `transfer b-21 solar_sail 2` + `launch_solar_sail b-21 --count 1` 返回 `OK`
  - `transfer b-20 small_carrier_rocket 1` + `launch_rocket b-20 sys-1 --layer 0 --count 1` 返回 `OK`
  - `set_ray_receiver_mode b-22 power` 返回 `OK`
  - `set_ray_receiver_mode b-22 photon` 在未解锁 `dirac_inversion` 时返回明确门禁错误：`photon mode requires dirac_inversion`
  - `jammer_tower`、`sr_plasma_turret`、`planetary_shield_generator`、`self_evolution_lab` 已可通过 `build` 进入真实建筑实例；其中：
    - `planetary_shield_generator` 的 `inspect` 运行时带 `shield.current_charge`
    - `self_evolution_lab` 在补电后可进入 `running`

## 当前新发现的问题

### 问题 1：默认新局科研起点锁死，`config-dev + map.yaml` 无法合法完成第一门 `electromagnetism`

#### 复现

1. 启动隔离默认新局 `18121`
2. 登录 `p1`
3. 执行：
   - `summary`
   - `start_research electromagnetism`
   - `build 4 2 matrix_lab`
   - `build 4 2 wind_turbine`
4. 补查：
   - `GET /state/summary`
   - `GET /catalog`

#### 实际现象

- `summary` / `GET /state/summary` 显示默认新局当前只完成：
  - `dyson_sphere_program`
- 直接执行：
  - `start_research electromagnetism`
- 返回：
  - `VALIDATION_FAILED: at least one running research lab is required`
- 继续尝试：
  - `build 4 2 matrix_lab`
- 返回：
  - `VALIDATION_FAILED: building type matrix_lab requires research to unlock`
- 再尝试：
  - `build 4 2 wind_turbine`
- 返回：
  - `VALIDATION_FAILED: building type wind_turbine requires research to unlock`
- 同时 `/catalog` 显示：
  - `electromagnetism.cost = electromagnetic_matrix x10`
  - `electromagnetic_matrix` 这项科技才会解锁 `matrix_lab`

#### 结论

- 当前默认新局存在一条硬循环：
  - 要研究 `electromagnetism`，先要有 `running` 研究站
  - 要有研究站，需要先解锁 `matrix_lab`
  - 要解锁 `matrix_lab`，又要先完成 `electromagnetism` 之后的 `electromagnetic_matrix`
- 在当前默认新局里，玩家没有任何合法公开路径打破这条循环。

#### 影响

- `config-dev.yaml + map.yaml` 实际上不能像 `docs/player/玩法指南.md` 所说那样“从开局一路推进到中期工业化”
- 默认新局的：
  - 供电
  - 采矿
  - 科研
  - 工业化主线
  都无法进入第一步
- 当前问题不是“文档少写一步”，而是 **默认新局的 DSP 科技树起点在玩法上尚未实现**
- `docs/player/玩法指南.md`、`docs/player/上手与验证.md` 目前对普通新局的表述会误导玩家，以为 fresh save 可以直接走完整起步流程

## 改动要求

### 1. 为默认新局恢复一条真实可玩的科研起步路线

- 在 brand-new 的 `config-dev.yaml + map.yaml` 新局中，玩家必须能只通过公开玩法操作，合法完成第一门关键科技
- 目标结果不是“文档换成去 midgame 场景验证”，而是：
  - 普通新局本身就能启动第一段科技树
- 可接受的实现方式可以调整，但结果必须满足：
  - 不依赖外部预制存档
  - 不依赖手工改配置临时塞科技
  - 不依赖隐藏内部命令

### 2. 不得把研究系统回退成旧版“抽象研究点自动完成”

- 本轮验证已经确认研究系统目前要求：
  - `running` 研究站
  - 真实矩阵库存
- 修复默认新局起点时，不能把这套真实矩阵规则再改回“零投入自动完成”
- 要修的是 **起步闭环**，不是把研究约束删掉

### 3. 统一默认新局、科技门禁与玩家指南的说法

- 如果默认新局的第一阶段需要：
  - 预置某个建筑
  - 预置某种 starter 物资
  - 预置某项已完成科技
  那就直接把这一事实落到实现与文档
- 不要继续保留“fresh save 能从 0 开始”与“实际必须去已解锁研究站场景验证”这两套相互冲突的口径
- 需要同步更新至少：
  - `docs/player/玩法指南.md`
  - `docs/player/上手与验证.md`
  - 若涉及启动配置 / API 语义变化，再更新：
    - `docs/dev/服务端API.md`
    - `docs/dev/客户端CLI.md`

## 验收标准

1. 使用仓库内默认 `config-dev.yaml + map.yaml` 启动全新隔离存档。
2. 玩家 `p1` 只通过公开命令和初始状态，能够合法进入第一段科研闭环，而不是一上来就被“缺研究站”锁死。
3. `electromagnetism` 能在默认新局中通过真实玩法完成。
4. 默认新局的前期供电 / 采矿 / 科研 / 工业化步骤能按修正后的玩家指南真实复现。
5. 本轮已确认可用的 midgame 链路不能回退：
   - `switch_active_planet`
   - `orbital_collector`
   - `launch_solar_sail`
   - `launch_rocket`
   - `set_ray_receiver_mode` 的 `power` 成功与 `photon` 科技门禁
