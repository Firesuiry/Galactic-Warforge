# T030 建筑功能挂载与基础运行参数

## 需求细节
- 建筑功能挂载机制：生产、采集、运输、仓储、发电、电网、研究等功能的统一挂载方式。
- 基础运行参数模型：功耗/发电、产能/吞吐、占格、连接点、维护成本等。
- 运行参数与功能的生命周期钩子（启停、状态变更、故障/欠压）。
- 数据结构与接口保持简洁，不引入兼容层。

## 前提任务
- 无

## 实现结果
- 运行参数补充维护成本字段 `maintenance_cost`（矿物/能量），并加入运行参数校验与客户端类型镜像。
- 建筑状态生命周期钩子以事件形式落地：新增 `building_state_changed` 事件，包含前后状态与原因；维护不足触发故障、欠压触发停机，资源恢复后自动回到运行态。
- 结算流程纳入维护成本消耗与欠压/故障判定，保持结构简洁且无兼容层。
- 更新服务端 API 文档的运行参数与事件说明。
