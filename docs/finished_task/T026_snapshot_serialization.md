# T026 Tick 级状态快照：数据模型与序列化

## 需求细节
- 定义 Tick 级快照的核心数据结构：世界/行星/建筑/物流/库存等可序列化状态。
- 提供快照序列化/反序列化接口，保证结构版本可扩展。
- 快照生成流程：从运行态聚合出快照数据的职责边界与依赖关系。
- 与命令驱动世界状态的接口对齐，保证快照可独立恢复。

## 前提任务
- T001 世界与地图
- T014 服务端接口约束与命令驱动

## 完成情况
- 新增 `snapshot` 领域模型：`Snapshot`/`WorldSnapshot`/`BuildingSnapshot`，覆盖 tick、地形、玩家、建筑、单位、资源与执行体信息，并保留建筑作业的 `prev_state` 用于独立恢复。
- 物流/库存运行态尚未落地，当前快照结构预留版本扩展点，待相关系统实现后补充字段与捕获逻辑。
- 提供快照序列化/反序列化接口（版本校验 + JSON 编解码），并实现从运行态捕获快照与从快照还原世界状态的流程。
- 为探索/发现状态加入 `DiscoverySnapshot`，支持与世界快照一起序列化并可独立恢复。
- 持久化层改为使用结构化快照类型，去除 `map[string]any` 的临时结构。
