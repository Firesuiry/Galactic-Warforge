# T025 多人审计与权限追踪

## 需求细节
- 审计日志：记录玩家/角色操作、目标与结果，支持按时间或用户检索。
- 权限追踪：标记命令来源与权限级别，支持后续审计验证。
- 审计留存策略：与日志/快照窗口联动的保留策略。
- 管理接口：导出审计记录或提供查询接口。

## 前提任务
- T014 服务端接口约束与命令驱动
- T024 命令日志与回滚

## 完成情况
- 新增审计模型与查询协议，支持按玩家、时间、权限等维度过滤，统一用于服务端与持久化层。
- 命令执行与预校验阶段写入审计日志，包含角色/命令来源/权限判断/目标与结果信息。
- 审计留存与快照窗口联动：快照裁剪时同步裁剪审计日志；回滚时同步裁剪超出 Tick 的审计数据。
- 新增审计查询接口 `GET /audit` 并更新服务端 API 文档。
