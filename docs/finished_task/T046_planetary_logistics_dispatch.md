# T046 行星内物流调度与供需匹配

## 需求细节
- 供需匹配：站点间匹配、优先级处理、容量约束。
- 调度策略：最短路或最低成本策略（基于已存在的距离/成本计算）。
- 派单生命周期：生成任务、锁定供需、完成回写。

## 交付物
- 服务端行星内物流调度实现。
- 单元测试或集成测试覆盖供需匹配、站点优先级、调度策略。
- 若涉及服务端 API 行为变更，更新 `docs/服务端API.md`。

## 实现结果
- 新增行星内物流调度：按站点输出优先级派单，匹配目标输入优先级并以曼哈顿距离选择最短路径，派单时锁定供需并扣减供给库存。
- 无人机落地时回写目标站点库存并刷新容量缓存，完成派单生命周期闭环。
- 回放与回滚流程补齐物流调度与相关结算步骤，保持与主 Tick 管线一致。
