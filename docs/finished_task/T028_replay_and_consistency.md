# T028 重放机制与一致性校验

## 需求细节
- 重放机制：从快照恢复后按 Tick 重放命令，保证可复现。
- 一致性校验：可选校验哈希/统计值，用于检测重放漂移。
- 重放控制接口：支持指定 Tick 范围、倍速/单步执行等调试开关。
- 与存档/回滚调用链路的依赖关系说明。

## 前提任务
- T026 Tick 级状态快照：数据模型与序列化
- T027 快照存储策略与保留窗口

## 实现结果
- 接入快照持久化：服务端启动即写入初始快照，Tick 结算时按策略保存快照并落盘。
- 新增重放引擎：从快照恢复世界与发现状态，按 Tick 重放命令日志与结算流程，支持范围/单步/倍速控制。
- 引入一致性摘要：生成世界摘要哈希与统计值，支持与目标 Tick 快照比对并输出漂移提示与命令结果差异计数。
- 增加快照检索能力：按 Tick 精确或向前查找快照，便于重放定位起点。
- 新增服务端接口 `POST /replay` 并更新服务端 API 文档。
- 依赖说明：重放入口依赖快照存储（T026/T027）提供的 Tick 快照作为恢复起点；命令日志驱动回放序列，可被后续存档/回滚链路复用为一致性校验与回滚重放基础。
