# T042 建筑接口与缓冲回退

## 需求细节
- 建筑输入/输出端口：与传送带/分拣器对接的端口模型与校验。
- 缓冲策略：建筑侧物品缓存的容量、优先级与回退策略。
- 回退与一致性：发生堵塞、溢出或被拒绝时的回滚与状态恢复。
- 运行时接口：建筑与物流系统交互的调用边界与数据结构。

## 交付物
- 服务端建筑与物流系统的接口与缓冲回退逻辑实现。
- 若涉及服务端 API 行为变更，更新 `docs/服务端API.md`。

## 实现结果
- 新增建筑 IO 结算：传送带与建筑 IO 口之间的输入/输出流转，支持端口容量与稳定顺序处理。
- 增加仓储预估接收与输出候选逻辑（`PreviewReceive`、`OutputCandidates`、`OutputQuantity`），用于接入物流前的容量评估。
- 新增 IO 端口偏移在占格内的校验，补齐端口参数合法性约束。
- 输入/输出过程增加回退：输入失败回滚至传送带前端，输出失败回退到输出缓冲/库存。
- 新增建筑 IO 结算单测，覆盖输入/输出、满仓回退、输出堵塞回退。
