## 当前问题列表

### 1. 开局最近的一批矿点超出默认执行体操作范围，玩法指南 B 阶段缺少“先移动执行体”的关键提示

- 复现：
  - 开局执行体 `u-2` 初始在 `(2,2)`，操作范围 `6`
  - 研究 `electromagnetism`
  - 直接尝试在最近钛矿 `planet-1-1-res-60` `(5,11)` 建 `mining_machine`
- 实际现象：
  - 命令会先被 `/commands` 接受
  - 下一 tick 的 `command_result` 返回：
    - `code = "OUT_OF_RANGE"`
    - `message = "executor out of range: 12 > 6"`
  - 实际必须先手动：
    - `move u-2 2 6`
    - `move u-2 2 10`
    才能继续把电塔接到 `(4,10)`，再在 `(3,11)` 建矿机
- 影响：
  - 玩家照着玩法指南从“风机 -> 电塔 -> 矿机”直走，会在第一处近矿点直接卡住
  - 这不是建筑或供电问题，而是玩法指南没有把“执行体移动”写成前置步骤

### 2. 原始事件接口仍然会被高频 tick 事件快速淹没，命令结果窗口很短

- 复现：
  - 连接 `GET /events/stream`，或查询 `GET /events/snapshot`
  - 不额外过滤高频事件
- 实际现象：
  - SSE / 快照里持续充满：
    - `resource_changed`
    - `threat_level_changed`
    - `tick_completed`
  - `command_result` 如果不是在发命令后立刻查询，很快就会被高频事件挤出窗口
  - 例如查询 `events/snapshot?since_tick=700` 时，本次试玩拿到的 `available_from_tick` 已经推进到 `2541`
- 影响：
  - CLI 默认静音高频事件后，普通玩家交互已经基本正常
  - 但 API 级排障和回溯体验仍然很差，尤其是想回头找几分钟前的命令失败原因时
