## 当前问题列表

- 本轮没有遗留阻断单星球物流试玩的问题。
- 本轮在浏览器实测中发现并当场修复了一个 Web 表单问题：
  - 现象：`物流槽位配置` 中，用户手动修改的 `item_id / mode / local_storage` 会被后续 `runtime` 刷新重置
  - 影响：按钮会重新变灰，导致错误物品被提交或根本无法按预期提交
  - 处理：已在 `client-web/src/features/planet-map/PlanetCommandPanel.tsx` 修复，并新增表单保持态测试
  - 回归：Playwright 复测后，`b-46` 已能稳定提交 `silicon_ore demand 60`
